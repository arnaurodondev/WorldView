## Findings

### Fixed ‚Äî MAJOR

#### F-001 ¬∑ WatchlistsTabPanel: `shadow-md` violates Terminal Dark design system
- **File**: `apps/worldview-web/components/portfolio/WatchlistsTabPanel.tsx:320`
- **Category**: DS-001 ‚Äî design-system violation
- **Issue**: Search results dropdown used `shadow-md` class. The Terminal Dark spec prohibits all `box-shadow`; elevation is conveyed by `border border-border` alone. Stale shadows make components look out of place in the Bloomberg-grade terminal layout.
- **Fix**: Removed `shadow-md`; left `border border-border` as the sole elevation signal.
- **Confidence**: HIGH

#### F-002 ¬∑ Portfolio service: `GET /v1/watchlists` returned soft-deleted watchlists
- **File**: `services/portfolio/src/portfolio/infrastructure/db/repositories/watchlist.py:45`
- **Category**: data-correctness / tenant data leakage
- **Issue**: `list_by_user` had no `status = 'active'` filter. Soft-deleted watchlists (`status = 'deleted'`) leaked through to the user-facing list endpoint. Logged as **Bug 1**.
- **Fix**: Added `WatchlistModel.status == "active"` predicate to `list_by_user` WHERE clause.
- **Live validation**: `GET /v1/watchlists` ‚Üí 4 active watchlists, 0 deleted rows ‚úì
- **Confidence**: HIGH (confirmed live)

#### F-003 ¬∑ Portfolio service: `POST /v1/watchlists` 500 on deleted-name reuse
- **File**: `services/portfolio/src/portfolio/infrastructure/db/repositories/watchlist.py:75`
- **Category**: error-handling / domain error translation
- **Issue**: After Bug 1 fix, the application-layer duplicate check only iterates `active` watchlists. A user who previously deleted a watchlist named "Tech" and tries to create a new "Tech" bypasses the use-case check, hits `save()`, and receives an `IntegrityError` from the DB unique constraint `uq_watchlists_user_name` (which covers ALL rows including deleted). This bubbled up as HTTP 500. Logged as **Bug 2**.
- **Fix**: Wrapped `session.flush()` in `save()` with `try/except IntegrityError`; translates `uq_watchlists_user_name` violations to `WatchlistAlreadyExistsError` (domain error globally mapped to HTTP 409).
- **Live validation**: `POST /v1/watchlists` reuse of a deleted name ‚Üí `409 WATCHLIST_ALREADY_EXISTS` ‚úì
- **Confidence**: HIGH (confirmed live)

#### F-004 ¬∑ Portfolio service: redundant `is_active()` guard in `CreateWatchlistUseCase`
- **File**: `services/portfolio/src/portfolio/application/use_cases/watchlist.py:99`
- **Category**: logic-error / misleading code
- **Issue**: After Bug 1 fix, `list_by_user` returns only active watchlists, making the per-item `is_active()` guard in `CreateWatchlistUseCase` permanently true. The comment next to the guard referenced a "Bug 1 fix" that hadn't been applied yet, creating misleading context about which invariant each layer was actually enforcing.
- **Fix**: Removed the `is_active()` predicate from the loop; added comment explaining the `list_by_user` active-only contract instead.
- **Confidence**: HIGH

### Fixed ‚Äî MINOR

#### F-005 ¬∑ `index-ticker-stale.test.tsx` failing on `searchInstruments` mock gap
- **File**: `apps/worldview-web/__tests__/index-ticker-stale.test.tsx:52`
- **Category**: test-isolation / mock completeness
- **Issue**: `IndexTicker.tsx` was refactored (unstaged working-tree change) to a two-step flow: Step 1 resolves ticker symbols to instrument_id UUIDs via `searchInstruments`; Step 2 batch-quotes the resolved UUIDs. The test mock factory provided `getBatchQuotes` but not `searchInstruments`. Without `searchInstruments`, `tickerToId` was never populated, `resolvedIds` was empty, the batch quote query was disabled, and all 4 test cases failed with prices showing "‚Äî" instead of expected values.
- **Fix**: Added `mockSearchInstruments` returning `{ results: [{ instrument_id: ticker, ... }] }` to the `createGateway` mock. Using `instrument_id: ticker` (same as ticker symbol) keeps lookups consistent with fixture keys.
- **Result**: 418/418 frontend tests pass ‚úì
- **Confidence**: HIGH

### Deferred ‚Äî MINOR
