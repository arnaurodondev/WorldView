## CRITICAL Issues — FIXED

### D-009 / DP-005 (2 agents, HIGH confidence) — FIXED in `5b0b9410`
- **File**: `services/nlp-pipeline/src/nlp_pipeline/infrastructure/messaging/consumers/article_consumer.py`
- **Issue**: `_emit_temporal_events` used `common.ids.new_uuid7()` (random) for the temporal event `event_id` AND passed no `event_id` kwarg to `outbox_repo.add()`. On Kafka re-delivery of the same article, `_emit_temporal_events` generated new UUIDs, bypassing the `ON CONFLICT (event_id) DO NOTHING` guard. Duplicate `intelligence.temporal_event.v1` events emitted downstream.
- **Fix**: Added loop index enumeration `for _idx, evt in enumerate(raw_events)`, derived `te_event_id = uuid.UUID(uuid5_from_parts(str(doc_id), raw_type, str(_idx)))`, and passed `event_id=te_event_id` to `outbox_repo.add()`. Matches the deterministic pattern already used in `_enqueue_enriched` and `_enqueue_signal_events`.
