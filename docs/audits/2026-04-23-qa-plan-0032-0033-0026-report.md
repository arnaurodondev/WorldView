## Phase 1: Specialist Review Findings (Fixed)

### BLOCKING Issues Fixed

**F-001 — `EntityMentionModel.resolution_outcome` type annotation mismatch**
- **File**: `services/nlp-pipeline/src/nlp_pipeline/infrastructure/nlp_db/models.py`
- **Issue**: `Mapped[str | None]` used for a `nullable=False` column (server_default="unresolved"). SQLAlchemy ORM raises `ArgumentError` at startup when reading existing rows.
- **Fix**: Changed to `Mapped[str]` matching the column's non-nullable constraint.

**F-002 — `KgUsageLogRepository.log()` missing `log_id` (Rule 6 violation)**
- **File**: `services/knowledge-graph/src/knowledge_graph/infrastructure/intelligence_db/repositories/llm_usage_log.py`
- **Issue**: Raw INSERT relied on DB-side `gen_random_uuid()` default. Violates Rule 6 (all IDs must use `common.ids.new_uuid7()`).
- **Fix**: Added `"log_id": str(new_uuid7())` to INSERT params with explicit `common.ids` import.

**F-003 — TC003 unsafe-fix regression: 34 SQLAlchemy ORM model files broken**
- **Files**: 34 ORM model files across all services
- **Issue**: `ruff --unsafe-fixes` moved `datetime`, `Decimal`, `uuid.UUID` to `TYPE_CHECKING` blocks. SQLAlchemy calls `typing.get_type_hints()` at class creation — these must be runtime imports. Caused `Could not resolve all types within mapped annotation: "Mapped[datetime]"` startup crash.
- **Fix**: Restored all affected imports to runtime scope; added `"TC003"` to `ruff.toml` global ignore list to prevent recurrence.

**F-004 — TC003 unsafe-fix regression: 203+ Pydantic/schema files broken**
- **Files**: 203 Pydantic model/schema/config files
- **Issue**: Same unsafe-fix moved `UUID`, `datetime`, `Decimal` to `TYPE_CHECKING` in Pydantic models. Pydantic v2 evaluates annotations at runtime → `PydanticUndefinedAnnotation: name 'UUID' is not defined`.
- **Fix**: Restored all affected imports; `TC003` global ignore confirmed in `ruff.toml`.

### CRITICAL Issues Fixed

**F-005 — Integration test JWT sub is non-UUID, breaking `CurrentUserIdDep`**
- **Files**: `services/alert/tests/integration/conftest.py`, `test_s7_s10_pipeline.py`, `test_fanout.py`, `test_websocket.py`
- **Issue**: `_make_system_jwt()` used `"sub": "integration-test-system"` (not a UUID). `CurrentUserIdDep` parses `request.state.user_id` as UUID → `ValueError` → 401 on all authenticated GET requests (BP-165).
- **Fix**: Added `INTEGRATION_USER_ID = "00000000-0000-0000-0000-000000000099"` constant; updated JWT `sub` to this UUID; updated tests that create alerts to use this UUID as the watcher user_id.

**F-006 — Portfolio integration tests fail with 503 (JWKS not loaded)**
- **File**: `services/portfolio/tests/conftest.py`
- **Issue**: `integration_client` fixture called `create_app()` without setting `PORTFOLIO_INTERNAL_JWT_SKIP_VERIFICATION=true`. `InternalJWTMiddleware` fails closed when JWKS public key is absent (no JWKS server in CI). Pre-existing from incomplete PLAN-0025 migration.
- **Fix**: Added `os.environ.setdefault("PORTFOLIO_INTERNAL_JWT_SKIP_VERIFICATION", "true")` at module level in `tests/conftest.py`. Also seeded `INTEGRATION_TENANT_ID`/`INTEGRATION_USER_ID` in the `integration_client` fixture so JWT-state routes find valid DB rows.

**F-007 — `content-store` InternalJWTMiddleware mypy `attr-defined` errors**
- **File**: `services/content-store/src/content_store/infrastructure/middleware/internal_jwt.py`
- **Issue**: `self.app.state._internal_jwt_public_key = key` fails mypy type check — `BaseHTTPMiddleware.app` is typed as `ASGIApp` (no `.state`).
- **Fix**: Added `# type: ignore[attr-defined]` on both assignment lines (lines 82, 112).

## Pre-Existing Issues (Out of Scope)

### Portfolio Integration Tests: 24 failures from PLAN-0025 Incomplete Migration

After PLAN-0025 added `InternalJWTMiddleware` to all services, the portfolio integration test fixtures were not fully updated. Specifically:

1. Tests create tenants/users via API and then operate on them with `X-Tenant-ID`/`X-Owner-ID` headers — but the routes now read identity from JWT `request.state` (not headers).
2. The JWT has a fixed `tenant_id = INTEGRATION_TENANT_ID`, but tests create entities under dynamically-created tenant IDs. The route's lookup (`users.get(user_id, INTEGRATION_TENANT_ID)`) finds no match → "USER_INACTIVE" error.

**Root cause**: The test fixture architecture assumes per-request JWT customization (different tenant_id per test) which requires a larger refactor of the integration test suite to use `make_jwt(tenant_id=X)` per-request signing. This is not in scope for PLAN-0032/0033/0026.

**Net effect**: This QA pass improved the situation from 43 failed (503 JWKS not loaded) to 24 failed (design mismatch). The 2 reverse-index tests specifically in PLAN-0032 scope now pass correctly.

**Recommendation**: Create PLAN-0034 to complete the PLAN-0025 integration test migration for portfolio service (update test fixtures to use per-test JWT signing or testcontainer-seeded identity).
