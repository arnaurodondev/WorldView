### GAP-4 · State F — Candlesticks + Drawing Tools + Amber Colors — QUALITY FAILURES

**Status**: Frame exists (sL0wd) but 3 verifiable quality failures.

#### GAP-4a: Candlestick structure incorrect
- **Current**: Filled rectangles only — no wick structure.
- **Required**: 3-node group per candle:
  - `wick-top`: 2px wide rect, height = high-to-body-top, full-opacity color
  - `body`: 12px wide rect, height = |open-close|, cornerRadius=1
  - `wick-bottom`: 2px wide rect, height = body-bottom-to-low, full-opacity color
  - Green candle (close > open): all nodes fill=`#26A69A`
  - Red candle (close < open): all nodes fill=`#EF5350`
- **Impact**: Chart looks like a bar chart, not a TradingView-quality OHLCV chart.

#### GAP-4b: Drawing tools sidebar (jayD2) — render verification needed
- Built via C() but icon text nodes may have opacity inherited from source node (WoVQh has opacity:0 on some children).
- Required: batch_get jayD2 subtree → verify each icon text node has opacity:1 and correct fill color.
- Expected icons: Crosshair `✛`, Trend `⟋`, H-Line `─`, Fib `Φ`, Rectangle `▭`, Text `T`

#### GAP-4c: Amber color verification for MA50 / price label
- MA50 line and current price label should use `#F0C040` (amber).
- Set via explicit hex in prior session — screenshot verification pending.
- If invisible: apply U() update with explicit fill hex on the MA50 frame and price text node.
