## CRITICAL — Fixed

### F-003 (CRITICAL) `useConfirmable` component identity race — useMemo with dialogOpen dep
**File**: `hooks/useConfirmable.tsx`
**Issue**: `BoundConfirmDialog` was constructed via `useMemo([..., dialogOpen])`. React treats a function defined inside useMemo as a new component type on each dep change — Radix Dialog unmounts and remounts on every open/close cycle, swallowing focus and the modal animation.
**Fix**: Changed to `useCallback` + `dialogOpenRef` (ref reads current open state without creating a new component type).
**Confidence**: HIGH

### F-004 (CRITICAL) Stale `isPending` closure in T1 timer
**File**: `hooks/useConfirmable.tsx`
**Issue**: `isPending` state snapshot in the `useCallback` closure meant a programmatic double-call (`execute(); execute()` within one render cycle) could pass the guard twice, orphaning the first AbortController.
**Fix**: Added `isPendingRef = useRef(false)` as a race-free guard; removed `isPending` from `useCallback` deps.
**Confidence**: HIGH

### F-005 (CRITICAL) False Undo affordance — toast dismissed after action fires
**File**: `hooks/useConfirmable.tsx`
**Issue**: The Undo toast remained visible after the timer expired. If the user clicked Undo after the action had already run, `undoTimerRef.current` was already null so `clearTimeout` was a no-op — but the visual "Undo" button was still clickable, suggesting the action could be reversed.
**Fix**: Added `toast.dismiss(toastId)` immediately before the deferred action fires inside the setTimeout callback.
**Confidence**: HIGH

### F-006 (CRITICAL) XSS via `javascript:` URLs in MarkdownContent
**File**: `components/ui/markdown-content.tsx`
**Issue**: An allowlist on `href` (`/^(https?:|mailto:)/`) was correct for external links but stripped relative paths like `/dashboard`, breaking ARIA link role. Worse, the first draft used no sanitization at all.
**Fix**: Switched to blocklist: `isDangerous = /^(javascript:|data:|vbscript:)/i.test(href.trim())`. Blocks executable schemes, passes https/http/mailto/relative/#anchors. WHY comment explains the allowlist-vs-blocklist tradeoff.
**Confidence**: HIGH

### F-007 (CRITICAL) ReadableStream lock — missing `reader.cancel()` on early exit paths
**File**: `features/chat/hooks/useChatStream.ts`
**Issue**: On `[DONE]`, error, or abort, the code returned from inside the `try` block without releasing the stream lock. Each early return left the connection pool slot occupied.
**Fix**: Moved `reader` declaration to before the `try` block; added `reader.cancel().catch(() => {})` in the `finally` block — runs on all exit paths.
**Confidence**: HIGH

## MAJOR — Fixed

### F-008 (MAJOR) Double-send race in `useChatStream` — state snapshot guard
**File**: `features/chat/hooks/useChatStream.ts`
**Issue**: `if (streaming)` guard in `useCallback` reads a stale state snapshot. Two rapid `send()` calls (or a programmatic double-call in tests) could both pass the guard before either sets streaming.
**Fix**: Added `isStreamingRef = useRef(false)` as a ref-based guard; changed guard to `if (isStreamingRef.current)`; removed `streaming` from `useCallback` deps array.
**Confidence**: HIGH

### F-009 (MAJOR) Raw `err.message` leaks internal details in chat error
**File**: `features/chat/hooks/useChatStream.ts`
**Issue**: `setChatError(err.message)` could expose internal hostnames, stack traces, or HTTP status text from reverse proxies into Sentry/Datadog.
**Fix**: Map known HTTP status codes to safe user-facing strings ("Session expired", "Server error", "Request failed"). Parse status code from the controlled error message (`Stream request failed: NNN …`).
**Confidence**: MEDIUM

### F-010 (MAJOR) Unsafe citation cast in `useChatStream`
**File**: `features/chat/hooks/useChatStream.ts`
**Issue**: `pendingCitations = data as Message["citations"]` — accepted any JSON shape from an SSE frame, including `javascript:` URLs that `CitationList` renders as `<a href>`.
**Fix**: Added runtime type guard: filter to objects with `url: string` matching `/^(https?:|mailto:)/i`.
**Confidence**: HIGH

### F-011 (MAJOR) `holdingId` URL injection risk
**File**: `lib/command-actions.ts` (action `portfolio.sell`)
**Issue**: `navigate(\`/portfolio?trade=sell&ticker=${row.ticker}&holding=${row.holdingId}\`)` — unencoded URL params. A ticker containing `&` or `=` (exotic OTC symbol) could corrupt the query string.
**Fix**: Added `encodeURIComponent()` around both `row.ticker` and `row.holdingId`.
**Confidence**: MEDIUM

### F-012 (MAJOR) Treemap FP drift — `shortSide <= 0` guard missing
**File**: `lib/treemap.ts`
**Issue**: Floating-point accumulated subtractions can reduce `shortSide` to a small negative value. Division by it produced Infinity coordinates, causing tail tiles to pile at origin (invisible but wasted DOM nodes).
**Fix**: Added `if (shortSide <= 0) break;` before the row-building loop. WHY comment explains the FP drift mechanism.
**Confidence**: HIGH

### F-013 (MAJOR) SSE guard race (removed `streaming` dep)
Covered by F-008 above (same fix).

## NITs — Fixed

### N-001 Misleading comment in `ConfirmDialog.handleConfirm`
**File**: `components/ui/confirm-dialog.tsx`
**Fix**: Clarified that dialog always closes immediately; async outcome surfaces via toast error pattern, not by keeping dialog open.

### N-002 Sonner mock missing `toast.dismiss` property
**File**: `__tests__/useConfirmable.test.ts`
**Issue**: Test mock `vi.mock("sonner", () => ({ toast: vi.fn() }))` created a plain function — `toast.dismiss()` threw "not a function" once F-005's fix added `toast.dismiss(toastId)`.
**Fix**: `Object.assign(vi.fn(), { dismiss: vi.fn() })` gives the mock function the same shape as sonner's real `toast`.

### N-003 Reader mock missing `cancel()` in useChatStream tests
**File**: `features/chat/hooks/__tests__/useChatStream.test.tsx`
**Issue**: `makeReader` and `makeAbortableReader` returned objects without `.cancel()`. F-007's fix calls `reader.cancel()` in `finally`, causing 4 tests to fail with "reader.cancel is not a function".
**Fix**: Added `cancel: vi.fn().mockResolvedValue(undefined)` to both factories and updated their type signatures.
