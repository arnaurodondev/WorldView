# Content Store Service (S5) — Agent Context

**Port**: 8005 | **DB**: `content_store_db` | **Status**: Compliant

## Mission
Consume raw articles. HTML cleaning (readability-lxml + bleach). 3-stage deduplication (URL hash, normalized hash, Valkey LSH MinHash). Canonical ID assignment. Clean text storage in MinIO.

## Domain Entities
Document (canonical), MinHashSignature, EntityMention, OutboxEvent

## Kafka Topics
| Direction | Topic | Notes |
|-----------|-------|-------|
| Produces | `content.article.stored.v1` | Article cleaned, deduped, canonical ID assigned |
| Consumes | `content.article.raw.v1` | Raw articles from S4 |

## Key Tables
documents, minhash_signatures, minhash_entity_mentions, outbox_events, dead_letter_queue

## API Endpoints (4)
healthz, readyz, metrics, articles query/detail

## Pitfalls
- `minhash_signatures.signature` must be `INTEGER[]`, NOT `BYTEA` (band-by-band Jaccard requires integer arithmetic)
- `minhash_entity_mentions.entity_id` is logical FK only (intelligence_db is separate DB — no Postgres-level constraint)
- Dedup thresholds: news hard=0.72, soft=0.55; filings hard=0.85
