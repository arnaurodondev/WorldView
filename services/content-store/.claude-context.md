# Content Store Service (S5) — Agent Context

**Port**: 8005 | **DB**: `content_store_db` | **Status**: Feature-complete

## Mission
Consume raw articles. HTML cleaning (readability-lxml + bleach). 3-stage deduplication (URL hash, normalized hash, Valkey LSH MinHash). Canonical ID assignment. Clean text storage in MinIO.

## Domain Entities
CanonicalDocument, MinHashSignature, EntityMention, DeduplicationDecision, DedupThresholds, CorroborationPolicy

## Domain Enums
DedupOutcome (6 values), DocumentStatus (5 values), OutboxStatus, ResolutionStatus, SourceType (5 values)

## Application Layer
- `text_cleaning/cleaner.py` — extract (HTML/XML/JSON/text), sanitize, normalize, clean pipeline
- `deduplication/stage_a_raw.py` — SHA-256 raw bytes hash + DB lookup
- `deduplication/stage_b_normalized.py` — URL normalization (strip UTM, sort params) + text hash
- `deduplication/minhash_compute.py` — normalize_financial_text, compute_shingles (word bigrams + char trigrams), compute_minhash (128 perms → list[int])
- `use_cases/process_article.py` — ProcessArticleUseCase: pipeline orchestrator (clean → dedup → silver → DB → outbox); LSH indexed by consumer post-commit (CR-3)

## Infrastructure Layer
- `infrastructure/storage/minio_silver.py` — Silver-tier canonical doc storage (JSON envelope)
- `infrastructure/messaging/consumers/article_consumer.py` — Kafka consumer for content.article.raw.v1
- `infrastructure/messaging/consumers/article_consumer_main.py` — Standalone consumer entry point (R22)
- `infrastructure/messaging/outbox/dispatcher.py` — Outbox dispatcher for content.article.stored.v1 (BP-001 compliant)
- `infrastructure/messaging/outbox/dispatcher_main.py` — Standalone dispatcher entry point (R22)

- `infrastructure/metrics/prometheus.py` — Prometheus counters, histograms, gauges (s5_*)
- `infrastructure/db/repositories/dlq.py` — DLQ repository (list, retry, resolve)

## MinIO Compensating GC (PLAN-0001-B-R3)
- `ProcessingSummary.minio_silver_key` is set when silver write succeeds; used by consumer for GC on commit failure
- `ArticleConsumer._handle_message` wraps `super()._handle_message(msg)` in try-except; on exception, deletes `_current_summary.minio_silver_key` from silver bucket
- GC fires before re-raising — the original exception is always preserved and re-raised
- Post-commit LSH indexing (`lsh.index()`) runs only on success path (after `super()._handle_message`)
- Test: `tests/unit/infrastructure/test_minio_gc.py`
- `_current_summary` is reset to `None` at the start of every `_handle_message` call (prevents stale GC from a previous message)

## QA-Discovered Pitfalls (PLAN-0001-B-R2)
- `move_to_dead_letter` must INSERT a DLQ row with original payload — not just update outbox status (BP-020)
- LSH `index()` must happen AFTER `session.commit()` — calling before commit creates phantom entries on rollback (CR-3)
- `outbox_events` migration DDL must match ORM exactly — add DDL alignment tests for any new service (BP-019)
- `requeue()` must use `entry.payload_json or {}` — not hardcoded empty dict
- DDL must never use `gen_random_uuid()` defaults on UUID PKs — all IDs are app-generated UUIDv7

## Pitfalls
- `minhash_signatures.signature` must be `INTEGER[]`, NOT `BYTEA` (band-by-band Jaccard requires integer arithmetic)
- `minhash_entity_mentions.entity_id` is logical FK only (intelligence_db is separate DB — no Postgres-level constraint)
- Dedup thresholds: news hard=0.72, soft=0.55; filings hard=0.85
- ASGI transport does NOT trigger lifespan — set app.state directly in unit test fixtures
- `readability-lxml>=0.8.1` (not ==0.8 which doesn't exist on PyPI)
- `pytest>=8.0` + `pytest-asyncio>=0.23.4` (avoid pinned ==8.0 + ==0.23.4 version conflict)
