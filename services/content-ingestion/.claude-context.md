# Content Ingestion Service (S4) — Agent Context

**Port**: 8004 | **DB**: `content_ingestion_db` | **Status**: Compliant

## Mission
Scheduled polling of EODHD news, SEC EDGAR filings, Finnhub, NewsAPI. Domain allowlists, rate limiting per source, relay fallback, raw payload storage to MinIO.

## Domain Entities
Source, FetchResult, RawArticle, TokenBucket, FetchSummary

## Kafka Topics
| Direction | Topic | Notes |
|-----------|-------|-------|
| Produces | `content.article.raw.v1` | Raw article fetched event (via outbox) |

sources, source_adapter_state, article_fetch_log, outbox_events, dead_letter_queue

## Prometheus Metrics
- `s4_fetches_total{source, status}` — fetch operations counter
- `s4_fetch_duration_seconds{source}` — fetch cycle duration histogram
- `s4_outbox_pending_total` — pending outbox events gauge
- `s4_dlq_total` — open DLQ entries gauge

## Scheduler
- `IngestionScheduler` with `IntervalTrigger` per enabled source
- PostgreSQL advisory lock (`pg_try_advisory_lock`) for single-leader
- `ADAPTER_REGISTRY` maps SourceType → adapter class

## Pitfalls
- All TIMESTAMP columns must be TIMESTAMPTZ (not TIMESTAMP)
- DB + Kafka writes must be same transaction (outbox pattern)
- Always use `common.ids.new_uuid7()` for PKs, not `uuid.uuid4()`
- Always use `common.time.utc_now()`, never `datetime.now()` or `datetime.utcnow()`
- Mocking `asyncio.to_thread` must patch at call site
- FastAPI route files must NOT use `from __future__ import annotations` (breaks DI resolution)
- TokenBucket requires all 4 args: capacity, tokens, refill_rate, last_refill

## Bug Patterns
- BP-001: Use `OutboxEventValueSerializer`, not `KafkaEventValueSerializer`
- BP-004: Alembic migration target issues
- BP-005: PostgreSQL TIMESTAMPTZ semantics
- BP-009: Let factory derive DispatcherConfig from settings
