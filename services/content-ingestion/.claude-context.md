# Content Ingestion Service (S4) — Agent Context

**Port**: 8004 | **DB**: `content_ingestion_db` | **Status**: Compliant

## Mission
Scheduled polling of EODHD news, SEC EDGAR filings, Finnhub, NewsAPI, and Polymarket prediction markets. Domain allowlists, rate limiting per source, relay fallback, raw payload storage to MinIO.

## Process Architecture (R22 — PLAN-0006)
4 independent processes, same Dockerfile, different `command` overrides:

| Process | Entry Point | Description |
|---------|------------|-------------|
| API | `uvicorn content_ingestion.app:create_app --factory` | HTTP endpoints only — no background processing |
| Scheduler | `python -m content_ingestion.infrastructure.scheduler.scheduler_process` | Tick-based: evaluates sources, creates task rows |
| Worker | `python -m content_ingestion.infrastructure.workers.worker` | Claims tasks, executes fetch-and-write pipeline |
| Dispatcher | `python -m content_ingestion.infrastructure.messaging.outbox.dispatcher_main` | Publishes outbox events to Kafka |

### Scheduler
- `SchedulerProcess` → `ScheduleDueSourcesUseCase` on each tick
- Idempotent: `ON CONFLICT (source_id, window_start) DO NOTHING`
- Config: `scheduler_tick_interval_seconds` (60s), `scheduler_max_tasks_per_tick` (100)

### Worker
- `WorkerProcess` → `ClaimTasksUseCase` → `ExecuteContentTaskUseCase` (for standard sources)
- POLYMARKET tasks routed to `_execute_polymarket_task()` → `PolymarketAdapter` → `FetchAndWritePredictionMarketsUseCase`
- Semaphore-gated concurrency (default 2), task timeout (120s)
- Back-pressure: sleeps when queue is empty
- Session optimization (R24): no session held during external API calls

### Legacy Scheduler (deprecated)
- `IngestionScheduler` in `infrastructure/scheduler/scheduler.py` — APScheduler-based, still in codebase
- `ADAPTER_REGISTRY` maps SourceType → adapter class (still used by worker)

## Advisory Lock & Watermarks
- Advisory lock from shared `messaging.pg.advisory_lock` (SHA-256 hashing, deterministic)
- Lock held only during DB writes, NOT during external API fetches (prevents pool exhaustion)
- Watermarks: `source_adapter_state.last_watermark` drives incremental polling via `from_date` parameter
- Adapters accept `from_date` kwarg which overrides `source.config["from_date"]`

## Auth
- `admin_token` — admin-only operations (source CRUD, DLQ, status)
- `internal_service_token` — shared across services (`INTERNAL_SERVICE_TOKEN` env var)
- Both use `hmac.compare_digest()` for timing-safe comparison

## Dispatcher
- Standalone process (`dispatcher_main.py`) with SIGINT/SIGTERM signal handling
- Uses `_build_factories()` for its own session factory (R22)
- No longer embedded in API lifespan (PLAN-0006 Wave B-4)

## Pitfalls
- All TIMESTAMP columns must be TIMESTAMPTZ (not TIMESTAMP)
- DB + Kafka writes must be same transaction (outbox pattern)
- Always use `common.ids.new_uuid7()` for PKs, not `uuid.uuid4()`
- Always use `common.time.utc_now()`, never `datetime.now()` or `datetime.utcnow()`
- Mocking `asyncio.to_thread` must patch at call site
- FastAPI route files must NOT use `from __future__ import annotations` (breaks DI resolution)
- TokenBucket requires all 4 args: capacity, tokens, refill_rate, last_refill
- Never use Python `hash()` for advisory locks — use `hashlib.sha256` (BP-015)
- Outbox payload field names must match Avro schema exactly (BP-017)
- httpx.AsyncClient shared in lifespan, closed on shutdown — never per-request
- Session rollback after IntegrityError before continuing loop
- DLQ `move_to_dead_letter` stores `payload_json` for requeue
- PolymarketAdapter uses `fetch_log_exists_fn(market_id, snapshot_at)` — different dedup signature from standard `exists_fn(url_hash)` (BP-057: no DB session in adapter itself)
- POLYMARKET worker path bypasses `ExecuteContentTaskUseCase` — uses `_execute_polymarket_task()` directly in `WorkerProcess`
- `SECEdgarAdapter._is_market_hours()` requires UTC-aware datetime input; uses `zoneinfo.ZoneInfo` (never pytz)

## MinIO Compensating GC (PLAN-0001-B-R3)
- `BronzeStoragePort.delete_object(key)` removes a bronze object (best-effort orphan GC)
- `FetchAndWriteUseCase` tracks `pending_minio_keys` per batch; GC on any exception before commit
- GC failures are logged as WARNING and must not mask the original exception
- Pattern: append key after `put_object`, clear list after successful `commit_fn()`, loop-delete on failure
- Test: `tests/unit/application/test_minio_gc.py`

## QA-Discovered Pitfalls (PLAN-0001-B-R2)
- `doc_id` in outbox payload must be a per-article UUIDv7 (`common.ids.new_uuid7()`), NOT `source_id` — source_id identifies the polling source, not the individual document
- SSRF URL validation must resolve DNS hostnames via `socket.getaddrinfo()` and check ALL resolved IPs against private ranges — non-IP hostnames previously bypassed the check
- DDL must never use `gen_random_uuid()` defaults on UUID PKs — all IDs are app-generated UUIDv7
- DLQ `move_to_dead_letter` stores `payload_json` for requeue

## Bug Patterns
- BP-001: Use `OutboxEventValueSerializer`, not `KafkaEventValueSerializer`
- BP-004: Alembic migration target issues
- BP-005: PostgreSQL TIMESTAMPTZ semantics
- BP-009: Let factory derive DispatcherConfig from settings
- BP-015: Python `hash()` for cross-process coordination — use `hashlib.sha256`
- BP-016: Advisory lock spanning external I/O — lock only during writes
- BP-017: Outbox payload fields must match Avro schema field names exactly
- BP-018: Client constructor signatures must match wiring code

## Provider Settings Pattern (PLAN-0005 Wave A-2)
All 4 adapter clients (EODHD, Finnhub, NewsAPI, SECEdgar) now receive typed `provider_cfg`
sub-models injected at construction — no module-level constants. Pattern:
- Sub-models (`EODHDProviderSettings` etc.) inherit from `BaseModel` (NOT `BaseSettings`)
- Wiring in `app.py._run_fetch_cycle` passes `provider_cfg=settings.eodhd` etc.
- Architecture test `CFG-SETTINGS-BASE` allows `BaseModel` for compound names (nested pattern)
- httpx timeout: `settings.http_client.timeout_seconds` / `connect_timeout_seconds`
- TokenBucket: capacity from `settings.eodhd.rate_limit_per_second` (cast to `int`)

## ContentIngestionTask State Machine (PLAN-0006 Wave B-1)
- Entity: `domain/entities.py::ContentIngestionTask` — unit of work for scheduler-worker pattern
- States: PENDING → CLAIMED → RUNNING → SUCCEEDED / RETRY / FAILED
- `claim(worker_id, lease_seconds)`: PENDING/RETRY → CLAIMED
- `start()`: CLAIMED → RUNNING (increments attempt_count)
- `succeed()`: RUNNING → SUCCEEDED
- `fail(error)`: RUNNING → RETRY (if attempts left) or FAILED (terminal)
- Factory: `create_for_source(source, is_backfill, window_start)`
- Config: `db_url_read`, `scheduler_tick_interval_seconds`, `worker_batch_size`, `worker_lease_seconds`, `worker_concurrency`, etc.

## Dual Session Factory (PLAN-0006 Wave B-2)
- `_build_factories(settings)` returns `(engine, write_factory, read_factory)`
- When `db_url_read` is empty or same as `db_url`, read_factory reuses write_factory (no wasted connections)
- Write engine: `pool_size=10, max_overflow=20, pool_pre_ping=True, expire_on_commit=False`
- Read engine (when separate): `pool_size=20, max_overflow=30`
- `app.state.write_factory` and `app.state.read_factory` set in lifespan

## TaskRepository (PLAN-0006 Wave B-2)
- `infrastructure/db/repositories/task.py` — `TaskRepository` with 6 methods
- `claim_batch(worker_id, limit, lease_seconds)` — CTE + `UPDATE … RETURNING` with `FOR UPDATE SKIP LOCKED`
- `add_many_idempotent(tasks)` — `ON CONFLICT (source_id, window_start) DO NOTHING`
- `has_active_task(source_id)` — checks PENDING/CLAIMED/RUNNING (does NOT exclude expired leases — use `recover_expired_leases` to reset stalled tasks first)
- `recover_expired_leases(now, lease_timeout_seconds)` — resets CLAIMED/RUNNING tasks with `lease_expires < now - lease_timeout_seconds` to RETRY; returns count recovered
- `update_status(task_id, status, error_detail)` — status + error update
- `count_by_status()` — for Prometheus metrics

## Unit of Work (PLAN-0006 Wave B-2 + PLAN-0009 Wave B-2)
- Ports: `application/ports/unit_of_work.py` — `ReadOnlyUnitOfWork` ABC + `UnitOfWork(ReadOnlyUnitOfWork)` ABC
- Adapters: `infrastructure/db/unit_of_work.py` — `SqlaReadOnlyUnitOfWork` (read replica) + `SqlaUnitOfWork` (write)
- 6 repository properties: `tasks`, `sources`, `fetch_logs`, `outbox`, `adapter_state`, `dlq`
- `UnitOfWork` has `commit()`, `rollback()`, `on_commit()` — `ReadOnlyUnitOfWork` has none (R27)
- Read-only use cases MUST use `ReadOnlyUnitOfWork` to leverage read replica (R27)
- Wired in `app.py`: `app.state.uow_factory` (write) + `app.state.read_uow_factory` (read)
- Dependencies: `UoWDep` (write) + `ReadUoWDep` (read) + `BronzeStorageDep` (MinIO)

## ExecuteContentTaskUseCase (PLAN-0009 Wave C-1)
- Keyword-only constructor with 7 injected dependencies — zero infrastructure imports (R25)
- `adapter_state_factory`, `fetch_log_factory`, `outbox_factory`: session → port callables (wired by worker)
- `bronze`: pre-built `BronzeStoragePort` (injected by worker)
- `adapter_builder`: `(SourceType, exists_fn) → SourceAdapterPort` (worker's `_build_adapter`)
- Metrics recording (`record_fetch`) moved to worker — called post-execute on success
- Worker's `_build_adapter` encapsulates all client construction, rate-limiter setup, and adapter registry lookup
