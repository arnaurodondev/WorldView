**Port**: 8004 | **DB**: `content_ingestion_db` | **Status**: Compliant

## Mission
Scheduled polling of EODHD news, SEC EDGAR filings, Finnhub, NewsAPI. Domain allowlists, rate limiting per source, relay fallback, raw payload storage to MinIO.

## Prometheus Metrics
- `s4_fetches_total{source, status}` — fetch operations counter
- `s4_fetch_duration_seconds{source}` — fetch cycle duration histogram
- `s4_outbox_pending_total` — pending outbox events gauge
- `s4_dlq_total` — open DLQ entries gauge

## Scheduler
- `IngestionScheduler` with `IntervalTrigger` per enabled source
- Exponential backoff on persistent failures: `min(interval * 2^failures, max_backoff)`, reset on success
- Hot-add/remove sources at runtime via `scheduler.add_source()` / `scheduler.remove_source()`
- `ADAPTER_REGISTRY` maps SourceType → adapter class

## Advisory Lock & Watermarks
- Advisory lock from shared `messaging.pg.advisory_lock` (SHA-256 hashing, deterministic)
- Lock held only during DB writes, NOT during external API fetches (prevents pool exhaustion)
- Watermarks: `source_adapter_state.last_watermark` drives incremental polling via `from_date` parameter
- Adapters accept `from_date` kwarg which overrides `source.config["from_date"]`

## Auth
- `admin_token` — admin-only operations (source CRUD, DLQ, status)
- `internal_service_token` — shared across services (`INTERNAL_SERVICE_TOKEN` env var)
- Both use `hmac.compare_digest()` for timing-safe comparison

## Dispatcher
- Supervised background task with restart-on-crash and exponential backoff
- `/readyz` reports "degraded" when dispatcher is down (`app.state.dispatcher_healthy`)

## Pitfalls
- All TIMESTAMP columns must be TIMESTAMPTZ (not TIMESTAMP)
- DB + Kafka writes must be same transaction (outbox pattern)
- Always use `common.ids.new_uuid7()` for PKs, not `uuid.uuid4()`
- Always use `common.time.utc_now()`, never `datetime.now()` or `datetime.utcnow()`
- Mocking `asyncio.to_thread` must patch at call site
- FastAPI route files must NOT use `from __future__ import annotations` (breaks DI resolution)
- TokenBucket requires all 4 args: capacity, tokens, refill_rate, last_refill
- Never use Python `hash()` for advisory locks — use `hashlib.sha256` (BP-015)
- Outbox payload field names must match Avro schema exactly (BP-017)
- httpx.AsyncClient shared in lifespan, closed on shutdown — never per-request
- Session rollback after IntegrityError before continuing loop
- DLQ `move_to_dead_letter` stores `payload_json` for requeue

## Bug Patterns
- BP-001: Use `OutboxEventValueSerializer`, not `KafkaEventValueSerializer`
- BP-004: Alembic migration target issues
- BP-005: PostgreSQL TIMESTAMPTZ semantics
- BP-009: Let factory derive DispatcherConfig from settings
- BP-015: Python `hash()` for cross-process coordination — use `hashlib.sha256`
- BP-016: Advisory lock spanning external I/O — lock only during writes
- BP-017: Outbox payload fields must match Avro schema field names exactly
- BP-018: Client constructor signatures must match wiring code
