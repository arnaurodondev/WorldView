# Content Ingestion Service (S4) ‚Äî Agent Context

**Port**: 8004 | **DB**: `content_ingestion_db` | **Status**: Compliant

## Mission
Scheduled polling of EODHD global news, EODHD per-ticker news (`eodhd_ticker_news`, auto-created per equity by `TickerNewsSymbolSyncWorker`), SEC EDGAR filings, Finnhub, NewsAPI, and Polymarket prediction markets. Domain allowlists, rate limiting per source, fetch-level dedup, raw payload storage to MinIO bronze. Also consumes `nlp.document.ready.v1` to flip tenant uploads to `ready`. `SourceType` = `contracts.enums.ContentSourceType` (eodhd, eodhd_ticker_news, sec_edgar, finnhub, newsapi, polymarket, manual).

## Process Architecture (R22 ‚Äî PLAN-0006)
5 independent processes, same Dockerfile, different `command` overrides:

| Process | Entry Point | Description |
|---------|------------|-------------|
| API | `uvicorn content_ingestion.app:create_app --factory` | HTTP endpoints only ‚Äî no background processing |
| Scheduler | `python -m content_ingestion.infrastructure.scheduler.scheduler_main` | Tick-based (`SchedulerProcess`): evaluates sources, creates task rows; also runs `TickerNewsSymbolSyncWorker` |
| Worker | `python -m content_ingestion.infrastructure.workers.worker_main` | Claims tasks, executes fetch-and-write pipeline |
| Dispatcher | `python -m content_ingestion.infrastructure.messaging.outbox.dispatcher_main` | Publishes outbox events to Kafka |
| Document-ready consumer | `python -m content_ingestion.infrastructure.messaging.consumers.document_ready_consumer_main` | Consumes `nlp.document.ready.v1`, marks tenant uploads `ready` |

### Scheduler
- `SchedulerProcess` ‚Üí `ScheduleDueSourcesUseCase` on each tick
- Idempotent: `ON CONFLICT (source_id, window_start) DO NOTHING`
- Config: `scheduler_tick_interval_seconds` (60s), `scheduler_max_tasks_per_tick` (100)

### Worker
- `WorkerProcess` ‚Üí `ClaimTasksUseCase` ‚Üí `ExecuteContentTaskUseCase` (for standard sources)
- POLYMARKET tasks routed to `_execute_polymarket_task()` ‚Üí `PolymarketAdapter` ‚Üí `FetchAndWritePredictionMarketsUseCase`
- Semaphore-gated concurrency (default 2), task timeout (120s)
- Back-pressure: sleeps when queue is empty
- Session optimization (R24): no session held during external API calls

### Legacy Scheduler (deprecated)
- `IngestionScheduler` in `infrastructure/scheduler/scheduler.py` ‚Äî APScheduler-based, still in codebase
- `ADAPTER_REGISTRY` (same module) maps SourceType ‚Üí adapter class (still used by `worker_main`): EODHD, EODHD_TICKER_NEWS, SEC_EDGAR, FINNHUB, NEWSAPI. POLYMARKET is intentionally absent (routed to `_execute_polymarket_task()`).

## Advisory Lock & Watermarks
- Advisory lock from shared `messaging.pg.advisory_lock` (SHA-256 hashing, deterministic)
- Lock held only during DB writes, NOT during external API fetches (prevents pool exhaustion)
- Watermarks: `source_adapter_state.last_watermark` drives incremental polling via `from_date` parameter
- Adapters accept `from_date` kwarg which overrides `source.config["from_date"]`
- **EODHD news batch sweep (QUOTA-OPT 2026-06-16)**: EODHD `/api/news` bills a FLAT 5 credits + 5/ticker PER REQUEST regardless of article count, so one request pulls the WHOLE batch since the watermark. Adapters build `from = watermark - eodhd.news_watermark_overlap_days`, `to = today`, `limit = eodhd.news_page_limit` (1000, EODHD max), and paginate via `offset` ONLY when a page comes back full ‚Äî a normal incremental run is ONE call. First run / empty watermark uses a BOUNDED backfill (`today - backfill_initial_days`), never "since epoch". After a non-empty sweep, `execute_task._newest_published_at()` advances `last_watermark` to the NEWEST article's `published_at` (not wall-clock `now`). On HTTP 429 the adapter raises `ProviderRateLimited`/`AdapterError` (retryable, NOT fatal) ‚Üí task RETRY with back-off and watermark NOT advanced. Intra-sweep url_hash dedup + downstream `FetchAndWriteUseCase` dedup absorb the overlap re-fetch.

## Auth
- `InternalJWTMiddleware` (RS256, JWKS from `{api_gateway_url}/internal/jwks`) gates ALL routes except skip paths (`/healthz`, `/readyz`, `/internal/v1/health`, `/health*`, `/metrics`) ‚Äî every `/api/v1/*`, `/admin/*`, `/internal/v1/ingest/submit` needs a valid `X-Internal-JWT`. Fails CLOSED (401) when JWKS key unavailable unless `internal_jwt_skip_verification=true` (never prod).
- `admin_token` (`X-Admin-Token`, `CONTENT_INGESTION_ADMIN_TOKEN`) ‚Äî admin routes (source CRUD, DLQ, status) require this IN ADDITION to the JWT gate (`AdminAuthDep` ‚Üí `verify_admin_token`, `hmac.compare_digest()`).
- Tenant document API derives `tenant_id`/`user_id` from JWT claims (no separate header token).

## Pitfalls
- All TIMESTAMP columns must be TIMESTAMPTZ (not TIMESTAMP)
- DB + Kafka writes must be same transaction (outbox pattern)
- Always use `common.ids.new_uuid7()` for PKs, not `uuid.uuid4()`
- Always use `common.time.utc_now()`, never `datetime.now()` or `datetime.utcnow()`
- Mocking `asyncio.to_thread` must patch at call site
- FastAPI route files must NOT use `from __future__ import annotations` (breaks DI resolution)
- TokenBucket requires all 4 args: capacity, tokens, refill_rate, last_refill
- Never use Python `hash()` for advisory locks ‚Äî use `hashlib.sha256` (BP-015)
- Outbox payload field names must match Avro schema exactly (BP-017)
- httpx.AsyncClient shared in lifespan, closed on shutdown ‚Äî never per-request
- Session rollback after IntegrityError before continuing loop
- DLQ `move_to_dead_letter` stores `payload_json` for requeue
- PolymarketAdapter uses `fetch_log_exists_fn(market_id, snapshot_at)` ‚Äî different dedup signature from standard `exists_fn(url_hash)` (BP-057: no DB session in adapter itself)
- POLYMARKET worker path bypasses `ExecuteContentTaskUseCase` ‚Äî uses `_execute_polymarket_task()` directly in `WorkerProcess`
- `SECEdgarAdapter._is_market_hours()` requires UTC-aware datetime input; uses `zoneinfo.ZoneInfo` (never pytz)

## MinIO Compensating GC (PLAN-0001-B-R3)
- `BronzeStoragePort.delete_object(key)` removes a bronze object (best-effort orphan GC)
- `FetchAndWriteUseCase` tracks `pending_minio_keys` per batch; GC on any exception before commit
- GC failures are logged as WARNING and must not mask the original exception
- Pattern: append key after `put_object`, clear list after successful `commit_fn()`, loop-delete on failure
- Test: `tests/unit/application/test_minio_gc.py`

## QA-Discovered Pitfalls (PLAN-0001-B-R2)
- `doc_id` in outbox payload must be a per-article UUIDv7 (`common.ids.new_uuid7()`), NOT `source_id` ‚Äî source_id identifies the polling source, not the individual document
- SSRF URL validation must resolve DNS hostnames via `socket.getaddrinfo()` and check ALL resolved IPs against private ranges ‚Äî non-IP hostnames previously bypassed the check
- DDL must never use `gen_random_uuid()` defaults on UUID PKs ‚Äî all IDs are app-generated UUIDv7
- DLQ `move_to_dead_letter` stores `payload_json` for requeue

## Provider Settings Pattern (PLAN-0005 Wave A-2)
All adapter clients (EODHD, EODHD ticker-news, Finnhub, NewsAPI, SECEdgar, Polymarket) receive typed `provider_cfg`
sub-models injected at construction ‚Äî no module-level constants. Pattern:
- Sub-models (`EODHDProviderSettings` etc.) inherit from `BaseModel` (NOT `BaseSettings`)
- Wiring in `app.py._run_fetch_cycle` passes `provider_cfg=settings.eodhd` etc.
- Architecture test `CFG-SETTINGS-BASE` allows `BaseModel` for compound names (nested pattern)
- httpx timeout: `settings.http_client.timeout_seconds` / `connect_timeout_seconds`
- TokenBucket: capacity from `settings.eodhd.rate_limit_per_second` (cast to `int`)

## ContentIngestionTask State Machine (PLAN-0006 Wave B-1)
- Entity: `domain/entities.py::ContentIngestionTask` ‚Äî unit of work for scheduler-worker pattern
- States: PENDING ‚Üí CLAIMED ‚Üí RUNNING ‚Üí SUCCEEDED / RETRY / FAILED
- `claim(worker_id, lease_seconds)`: PENDING/RETRY ‚Üí CLAIMED
- `start()`: CLAIMED ‚Üí RUNNING (increments attempt_count)
- `succeed()`: RUNNING ‚Üí SUCCEEDED
- `fail(error)`: RUNNING ‚Üí RETRY (if attempts left) or FAILED (terminal)
- Factory: `create_for_source(source, is_backfill, window_start)`
- Config: `db_url_read`, `scheduler_tick_interval_seconds`, `worker_batch_size`, `worker_lease_seconds`, `worker_concurrency`, etc.

## Dual Session Factory (PLAN-0006 Wave B-2)
- `_build_factories(settings)` returns `(engine, write_factory, read_factory)`
- When `db_url_read` is empty or same as `db_url`, read_factory reuses write_factory (no wasted connections)
- Write engine: `pool_size=10, max_overflow=20, pool_pre_ping=True, expire_on_commit=False`
- Read engine (when separate): `pool_size=20, max_overflow=30`
- `app.state.write_factory` and `app.state.read_factory` set in lifespan

## TaskRepository (PLAN-0006 Wave B-2)
- `infrastructure/db/repositories/task.py` ‚Äî `TaskRepository` with 6 methods
- `claim_batch(worker_id, limit, lease_seconds)` ‚Äî CTE + `UPDATE ‚Ä¶ RETURNING` with `FOR UPDATE SKIP LOCKED`
- `add_many_idempotent(tasks)` ‚Äî `ON CONFLICT (source_id, window_start) DO NOTHING`
- `has_active_task(source_id)` ‚Äî checks PENDING/CLAIMED/RUNNING (does NOT exclude expired leases ‚Äî use `recover_expired_leases` to reset stalled tasks first)
- `recover_expired_leases(now, lease_timeout_seconds)` ‚Äî resets CLAIMED/RUNNING tasks with `lease_expires < now - lease_timeout_seconds` to RETRY; returns count recovered
- `update_status(task_id, status, error_detail)` ‚Äî status + error update
- `count_by_status()` ‚Äî for Prometheus metrics

## Unit of Work (PLAN-0006 Wave B-2 + PLAN-0009 Wave B-2)
- Ports: `application/ports/unit_of_work.py` ‚Äî `ReadOnlyUnitOfWork` ABC + `UnitOfWork(ReadOnlyUnitOfWork)` ABC
- Adapters: `infrastructure/db/unit_of_work.py` ‚Äî `SqlaReadOnlyUnitOfWork` (read replica) + `SqlaUnitOfWork` (write)
- 6 repository properties: `tasks`, `sources`, `fetch_logs`, `outbox`, `adapter_state`, `dlq`
- `UnitOfWork` has `commit()`, `rollback()`, `on_commit()` ‚Äî `ReadOnlyUnitOfWork` has none (R27)
- Read-only use cases MUST use `ReadOnlyUnitOfWork` to leverage read replica (R27)
- Wired in `app.py`: `app.state.uow_factory` (write) + `app.state.read_uow_factory` (read)
- Dependencies: `UoWDep` (write) + `ReadUoWDep` (read) + `BronzeStorageDep` (MinIO)

## ExecuteContentTaskUseCase (PLAN-0009 Wave C-1)
- Keyword-only constructor with 7 injected dependencies ‚Äî zero infrastructure imports (R25)
