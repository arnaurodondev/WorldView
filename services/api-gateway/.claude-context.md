# API Gateway Service (S9) — Agent Context

**Port**: 8000 | **DB**: None (stateless) | **Status**: Compliant

## Mission
Unified API entry point, request routing, response composition (BFF pattern), auth/authz, rate limiting, per-endpoint caching, CORS, error standardization.

## Domain Entities
None — stateless proxy/compositor.

## Kafka Topics
None — stateless.

## API Endpoints
Proxies/composes all `/api/v1/*` endpoints from 6 backend services:
- Market Data (S3), Portfolio (S1), Content (S5), Intelligence (S6/S7), RAG (S8)
- 3 composition endpoints: company overview, portfolio holdings, bootstrap

## Critical Rules
- Frontend talks ONLY to this service — never directly to backends
- O(1) route lookup via dictionary (not regex/trie)
- Error responses always hide internals on 5xx (generic message)
- Fail-open on Valkey unavailable (don't block requests on cache failure)

## Pitfalls
- In-flight dedup uses `asyncio.Future` to prevent duplicate fetches
- Negative cache sentinel prevents cache stampede on transient errors
- Rate limit: 100 req/min authenticated, 20 req/min unauthenticated
- CORS must be configured for frontend origin
