# API Gateway Service (S9) — Agent Context

**Port**: 8000 | **DB**: None (stateless) | **Status**: Compliant

## Mission
Unified API entry point, request routing, response composition (BFF pattern), auth/authz, rate limiting, per-endpoint caching, CORS, error standardization.

## Domain Entities
None — stateless proxy/compositor.

## Kafka Topics
None — stateless.

## API Endpoints
Proxies/composes all `/api/v1/*` endpoints from 6 backend services:
- Market Data (S3), Portfolio (S1), Content (S5), Intelligence (S6/S7), RAG (S8)
- 3 composition endpoints: company overview, portfolio holdings, bootstrap

### Prediction Markets (PRD-0019, Wave C-1)

3 proxy routes under `/signals/prediction-markets` → S3 Market Data:

| Method | S9 Path | S3 Backend Path | Auth | Query Params |
|--------|---------|-----------------|------|--------------|
| GET | `/signals/prediction-markets` | `/api/v1/prediction-markets` | JWT | `status`, `limit`, `offset` |
| GET | `/signals/prediction-markets/{market_id}` | `/api/v1/prediction-markets/{id}` | JWT | — |
| GET | `/signals/prediction-markets/{market_id}/history` | `/api/v1/prediction-markets/{id}/history` | JWT | `from`, `to`, `limit` |

All 3 routes require JWT auth (via `_auth_headers(request)`) and add `X-Internal-Token` to backend calls.
S3 returns 404 if `market_id` is unknown; history raises 400 if `from_dt >= to_dt`.

## Critical Rules
- Frontend talks ONLY to this service — never directly to backends
- O(1) route lookup via dictionary (not regex/trie)
- Error responses always hide internals on 5xx (generic message)
- Fail-open on Valkey unavailable (don't block requests on cache failure)

## Auth Architecture (PRD-0025)
**DO NOT use the old `jwt_secret` / HS256 pattern.** PRD-0025 replaces it entirely:
- `OIDCAuthMiddleware` validates Zitadel RS256 JWTs via JWKS endpoint
- `InternalJWTIssuerMiddleware` signs RS256 internal JWTs with S9's private key
- Auth flow endpoints: `GET /v1/auth/login`, `GET /v1/auth/callback`, `POST /v1/auth/refresh`, `POST /v1/auth/logout`, `GET /v1/auth/me`
- JWKS endpoint: `GET /internal/jwks` (blocked from public by Traefik)
- Config vars: `OIDC_ISSUER_URL`, `OIDC_CLIENT_ID`, `OIDC_CLIENT_SECRET`, `OIDC_AUDIENCE`, `INTERNAL_JWT_PRIVATE_KEY`, `FRONTEND_URL`, `COOKIE_SECURE`

## Pitfalls
- In-flight dedup uses `asyncio.Future` to prevent duplicate fetches
- Negative cache sentinel prevents cache stampede on transient errors
- Rate limit: 100 req/min authenticated, 20 req/min unauthenticated
- CORS must be configured for frontend origin
