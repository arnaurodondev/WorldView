# API Gateway Service (S9) — Agent Context

**Port**: 8000 | **DB**: None (stateless) | **Status**: Compliant

## Mission
Unified API entry point, request routing, response composition (BFF pattern), auth/authz, rate limiting, per-endpoint caching, CORS, error standardization.

## Domain Entities
None — stateless proxy/compositor.

## Kafka Topics
None — no producer/consumer infra in `src/api_gateway/`. (Planned-only: an
`entity.dirtied.v1` consumer to invalidate the resolution cache — PLAN-0089 F2
step 6, not yet built.)

## Workers
One opt-in out-of-process worker: `python -m api_gateway.workers.bundle_prewarmer_main`
(PLAN-0099 R3) — keeps S7 intel/paths caches warm for hot entity IDs. Refuses to
start unless `API_GATEWAY_PREWARM_ENABLED=true` AND `API_GATEWAY_PREWARM_ENTITY_IDS`
has ≥1 UUID. Mints its own service-account RS256 JWT. Tuned by `PREWARM_*` env vars.

## API Endpoints
~186 route decorators proxying/composing from the backend services:
- S1 Portfolio (portfolios, holdings, transactions, watchlists, brokerage, feedback, risk-metrics)
- S3 Market Data (OHLCV, quotes, fundamentals, screener, prediction markets, heatmap, movers, peers/intraday/returns/levels)
- S5 Content Store (news: relevant, top, entity; article-metadata resolve)
- S6 NLP (full-text search, AI signals)
- S7 Knowledge Graph (entity graph, relations, contradictions, similar entities, narratives, paths, weird connections, economic calendar)
- S8 RAG/Chat (chat, threads, entity-context chat, briefings, proposals)
- S10 Alert (pending/ack/snooze/history alerts, email preferences)
- BFF composition use cases (`application/use_cases/`): company overview, instrument page-bundle, portfolio bundle, dashboard snapshot — plus market heatmap, top movers, sparklines, map layers
- 8 auth endpoints + `/internal/jwks` + `/internal/v1/service-token` + `/v1/admin/llm-costs`

### Route module map (proxy.py was split — PLAN-0089 B-3)
There is **no `routes/proxy.py`**. Routes live in per-domain modules combined by
`routes/__init__.py` into `main_router`: `alerts · chat (+chat_safety) · content ·
dashboard · instruments · intelligence · market · portfolio · signals`. Registered
separately in `app.py`: `health · internal · auth · admin_costs · risk_metrics`.
Shared proxy/auth helpers (`_auth_headers`, `_system_headers`, `proxy_json_response`)
live in `routes/helpers.py`. Retry-wrapped fetchers `_checked_get` (3× on 500/503) /
`_checked_post` (no retry) live in `clients/base.py` (and the legacy aggregate
`clients.py`). Per-domain typed httpx clients live in `clients/`.
**Registration order is load-bearing**: `signals_router` before `market_router`
(both declare `GET /v1/signals/ai`; the market.py copy is dead code); literal-segment
routes before parameterised ones.

### resolution.py (PRD-0089 F2)
`resolve_security_id(ident)` is the choke point that maps **ticker OR UUID →
canonical `instrument_id`**: UUID short-circuit → S3 `/instruments/lookup?symbol=` →
S7 KG `entities/lookup?ticker=` alias fallback → `InstrumentNotFoundError` (→404).
1h TTL cache; no Kafka invalidation yet (`entity.dirtied.v1` consumer is TODO).

### /api/v1 prefix compatibility
`app.py` registers a `strip_api_prefix` middleware rewriting `/api/v1/...` → `/v1/...`.
Both forms resolve to the same handler (Cloudflare/Next.js rewrite — Dashboard Regression #5).

Full endpoint reference: `docs/services/api-gateway.md`

### Prediction Markets (PRD-0019, Wave C-1)

3 proxy routes under `/signals/prediction-markets` → S3 Market Data:

| Method | S9 Path | S3 Backend Path | Auth | Query Params |
|--------|---------|-----------------|------|--------------|
| GET | `/signals/prediction-markets` | `/api/v1/prediction-markets` | JWT | `status`, `limit`, `offset` |
| GET | `/signals/prediction-markets/{market_id}` | `/api/v1/prediction-markets/{id}` | JWT | — |
| GET | `/signals/prediction-markets/{market_id}/history` | `/api/v1/prediction-markets/{id}/history` | JWT | `from`, `to`, `limit` |

All 3 routes require JWT auth (via `_auth_headers(request)`) and forward `X-Internal-JWT` to backend calls.
S3 returns 404 if `market_id` is unknown; history raises 400 if `from_dt >= to_dt`.

## Critical Rules
- Frontend talks ONLY to this service — never directly to backends
- Error responses always hide internals on 5xx (generic message)
- Rate limiting is **fail-closed** when Valkey is unconfigured/None → 503 (D-001). The OIDC user-cache read is independently fail-open (rebuilds identity from token claims). Do NOT "simplify" the limiter to fail-open.
- No Kafka, no DB, no SQLAlchemy in `src/api_gateway/` — stateless HTTP + Valkey only

## Auth Architecture (PRD-0025)
**DO NOT use the old `jwt_secret` / HS256 pattern.** PRD-0025 replaces it entirely:
- `OIDCAuthMiddleware` validates Zitadel RS256 JWTs via JWKS endpoint
- `InternalJWTIssuerMiddleware` signs RS256 internal JWTs with S9's private key
- Auth flow endpoints (8 total): `GET /v1/auth/login`, `GET /v1/auth/callback`, `POST /v1/auth/refresh`, `POST /v1/auth/logout`, `GET /v1/auth/me`, `GET /v1/auth/register`, `GET /v1/auth/ws-token`, `POST /v1/auth/dev-login`
- Dev-only endpoint: `POST /v1/auth/dev-login` — issues a test JWT for the seed demo user. Gated behind `oidc_config=None` (returns 403 when OIDC is configured). **NOT production-accessible.**
- Service-account endpoint: `POST /internal/v1/service-token` — PLAN-0057 Wave A-1 / BP-303. Background workers authenticate via shared `API_GATEWAY_SERVICE_ACCOUNT_TOKEN` secret + allow-listed `service_name` to mint a 5-minute RS256 JWT. Production-safe: NOT guarded by `app_env`. Allow-list (`routes/internal.py::_ALLOWED_SERVICE_NAMES`) currently holds **two** callers: `nlp-pipeline-price-impact` and `rag-chat-brief-scheduler` (PLAN-0094 W2 — morning-brief overnight scheduler). Constant-time secret comparison via `secrets.compare_digest`.
- JWKS endpoint: `GET /internal/jwks` (blocked from public by Traefik)
- Config vars (all `API_GATEWAY_`-prefixed): `OIDC_ISSUER_URL`, `OIDC_CLIENT_ID`, `OIDC_CLIENT_SECRET`, `OIDC_AUDIENCE`, `OIDC_DISCOVERY_OPTIONAL`, `INTERNAL_JWT_PRIVATE_KEY`, `INTERNAL_JWT_PUBLIC_KEY`, `JWT_KEY_VERSION`, `JWKS_GRACE_HOURS`, `FRONTEND_URL`, `COOKIE_SECURE`, the rate-limit tier vars (`RATE_LIMIT_REQUESTS`/`_FINANCIAL_MUTATION_REQUESTS`/`_UNAUTHENTICATED_REQUESTS`/`_PUBLIC_FEEDBACK_REQUESTS`), `SERVICE_ACCOUNT_TOKEN`, `DEEPINFRA_API_KEY`, and the `PREWARM_*` worker vars. Full table in `docs/services/api-gateway.md`.

- **SEC-003 (callback XSS)**: FIXED by PLAN-0032 Wave C-1 — `error`/`error_description` params in `/v1/auth/callback` now sanitized via RFC 6749 whitelist + regex before JSON reflection
- **SEC-004**: ADDRESSED BY PRD-0025 — RateLimitMiddleware wired
- **SEC-007**: ADDRESSED BY PRD-0025 — SecurityHeadersMiddleware added
- **SEC-008 (CORS port)**: FIXED by PLAN-0032 Wave C-1 — default `CORS_ORIGINS` uses port 3001 (worldview-web), not 3000

## Proxy Parameter Bugs (Fixed)
- **API-004 (transactions)**: `portfolio_id` is forwarded as `X-Portfolio-ID` header (not query param) to S1 — S1 reads it from the header
- **R-002 (economic-calendar)**: Proxy sends `event_type=economic` to S7 (not `type=economic`) — S7's `list_temporal_events` expects `event_type`

## Pitfalls
- In-flight dedup uses `asyncio.Future` to prevent duplicate fetches
- Negative cache sentinel prevents cache stampede on transient errors
- Rate limit tiers (wired from Settings in `app.py`): **2000 req/min authenticated read** (`RATE_LIMIT_REQUESTS`; in-code comments still cite the old 300), **30/min financial mutations** (POST/PUT/DELETE/PATCH on financial paths — separate bucket, PLAN-0094 W1), **10/min** `/v1/feedback/*`, low-rate **export** bucket (SEC-103), **20/min unauthenticated** per IP hash. Fail-closed (503) when Valkey unconfigured.
- CORS must be configured for frontend origin (port 3001 not 3000); set `API_GATEWAY_CORS_ORIGINS` for non-default ports
- `get_market_heatmap()` (`clients.py`) uses a SINGLE `GET /api/v1/market/sector-returns` call for ALL periods — the old path with 11 parallel screener `.post()` calls is dead code. Tests must mock `client.market_data.get`, not `.post`. See BP-465.
- `API_GATEWAY_ALERT_WS_URL` is separate from `API_GATEWAY_ALERT_URL` — the WS URL uses `ws://` scheme for WebSocket routing
- `dev-login` is blocked when `APP_ENV=production` regardless of OIDC config; also blocked when OIDC is configured (`oidc_config is not None`)
- `API_GATEWAY_SERVICE_ACCOUNT_TOKEN` must be set in production for background workers to use `POST /internal/v1/service-token`; without it the endpoint returns 503
- **PLAN-0112 weird-path proxies** (`routes/intelligence.py`, both → S7): `GET /v1/paths/between` (pairwise; key `pathbetween:{tenant}:{source}:{target}:{max_hops}:{limit}:{meaningful_only}`) + `GET /v1/connections/weird` (global feed; key `weird:{tenant}:{limit}:{offset}:{min_weirdness}:{since_days}:{entity_type}`). Both Valkey-cached 5 min, tenant-scoped cache keys (cross-tenant bleed guard); forward S7 status codes (400/404/422/503). `None` query params omitted from the upstream call so S7 applies its own defaults.
