## Pitfalls
- Alert preferences default to `enabled=True` when missing (synthesized defaults)
- Watchlist reverse-index cache is eventually consistent; invalidation uses DEL, not synchronous rebuild
- `WatchlistCacheDep` requires `app.state.valkey_client` or tests must mock it
- Watchlist soft-delete preserves rows in DB; consumers must check `status=deleted`
- Outbox status values: `pending` → `in_flight` → `published` (not `processing`/`delivered`)
- Idempotency key must be a valid UUID — malformed keys raise `IdempotencyKeyInvalidError` (422)
- `UnitOfWork.__aexit__` does NOT auto-commit (Option B / QA-006): every mutating use case must call `await uow.commit()` explicitly before returning
- Cache invalidation for watchlist members happens AFTER explicit `uow.commit()` (M-005: ordering fix)
- `PORTFOLIO_INTERNAL_SERVICE_TOKEN` must be set; an empty token causes 401 on all internal endpoints (startup warning logged)
- `DomainEvent.event_id` and `occurred_at` are plain strings (UUIDv7 hex and ISO-8601 Z) since Wave E3-4 (D-009)
- `InstrumentRef.id` uses `entity_id` from upstream event when available, for stable idempotent processing (M-017)
