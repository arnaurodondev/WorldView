## API Endpoints (27 public + 8 internal)

### Auth Model (PLAN-0025 Wave C)
- All requests: `InternalJWTMiddleware` validates `X-Internal-JWT` (RS256 from S9 JWKS).

## Provision Endpoint Logic (ProvisionUserUseCase)
4-step idempotent logic per PRD-0025 §3.3:
1. `find_by_external_id(sub)` → return existing (no writes)
2. `find_by_email_without_external_id(email)` → link + ACCOUNT_LINKED audit
3. `find_by_email_with_conflicting_external_id(email, sub)` → 409 + PROVISION_CONFLICT_409 audit
4. Neither → create Tenant + User atomically + USER_CREATED audit

## InternalJWTMiddleware
- Class: `portfolio.infrastructure.middleware.internal_jwt.InternalJWTMiddleware`
- Fetches JWKS from `{api_gateway_url}/internal/jwks` at startup (3 retries, 3s sleep)
- When `_public_key is None` (startup not called), passes through — test-safe
- Sets `request.state.tenant_id / user_id / role` from JWT claims
- Config: `api_gateway_url: str = "http://api-gateway:8000"`

## Pitfalls
- `InternalJWTMiddleware` only enforces auth when `_public_key` is not None — in unit tests lifespan is not called so middleware passes through
- `POST /tenants` now requires `role=system` — integration tests cannot use `POST /tenants` directly; use direct DB seeding
- `PORTFOLIO_INTERNAL_SERVICE_TOKEN` is REMOVED — do not reference it in new code
- Alert preferences default to `enabled=True` when missing (synthesized defaults)
- Watchlist soft-delete preserves rows in DB; consumers must check `status=deleted`
- `UnitOfWork.__aexit__` does NOT auto-commit: every mutating use case must call `await uow.commit()`
- `DomainEvent.event_id` and `occurred_at` are plain strings (UUIDv7 hex and ISO-8601 Z)
- `InstrumentRef.id` uses `entity_id` from upstream event when available (M-017). Post-F2 / PRD-0089 ([ADR-F-16](../../docs/architecture/decisions/ADR-F-16-instrument-entity-id-unification.md)): `entity_id == instrument_id` for `entity_type = 'financial_instrument'`, so the bridge field is always populated to the same UUID; the nullable branch is legacy-only.
- `PORTFOLIO_STORAGE_ACCESS_KEY` and `PORTFOLIO_STORAGE_SECRET_KEY` are required (no default) — service will not start without them
- Holdings are NOT updated by `RecordTransactionUseCase` (PLAN-0046) — only `UpsertHoldingsFromSnapshotUseCase` (brokerage sync) writes holdings. Manual transactions appear in history but don't update the holdings table until next sync.
- `GET /portfolios/{id}/value-history` default range is `date.min` to today (all snapshots), not 90 days — the "last 90 days" framing is UX only
- Concentration and holding-lots endpoints fetch live prices via `app.state.current_price_client` — the price client is wired at lifespan; `prices_stale=True` when individual quotes fail
- PLAN-0102 W2: `/internal/v1/users/{user_id}/portfolio/pnl` uses a separate `app.state.recent_prices_client` (HttpRecentPricesClient) which derives `last_close = price - price_change` from the S3 `/internal/v1/price/batch` response — both clients share the same `market_data_http` pool; do not re-use `current_price_client` for the P&L path (it doesn't expose `price_change`)
- BP-631 (PLAN-0103 W12): when verifying P&L endpoint shipped, distinguish FastAPI-default `{"detail":"Not Found"}` (route absent — image stale) from use case `{"detail":"User <uuid> not found ..."}` (route present, lookup failed). Validate route mounted with `docker exec worldview-portfolio-1 python -c "from portfolio.app import create_app; app=create_app(); print([r.path for r in app.routes if 'pnl' in r.path])"` — empty list = stale image, rebuild + recreate

## PLAN-0057 changes (News-Intelligence Pipeline Quality Repair, 2026-04-30 → 2026-05-01)

**New consumer / topic subscription**:
- Subscribes to `market.instrument.discovered.v1` (Wave D-2) — extends `InstrumentEventConsumer` to upsert an `InstrumentRef` from the placeholder discovered event. Allows Portfolio holdings/transactions to attach to instruments before market-data has fundamentals.
- The existing `market.instrument.created` subscription continues to populate full canonical metadata when fundamentals lands (now with CUSIP/FIGI/LEI/PrimaryTicker fields per Wave C-1 v3 schema).

**Pitfalls (added by this plan)**:
- F-DS-08 (deferred): on first deploy of `discovered.v1`, Kafka may replay backfilled events for ~10K-100K instruments. Single-partition serial consumer handles this in < 2 min; ON CONFLICT idempotency on InstrumentRef upsert prevents duplicates.
