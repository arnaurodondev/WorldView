# Market Ingestion Service (S2) — Agent Context

**Port**: 8002 | **DB**: `ingestion_db` | **Status**: Production-ready (migrations 0001–0025 shipped; head = `0025`)

## Mission
Scheduled polling of EODHD, Yahoo Finance, Finnhub, Polygon, and Alpaca. Raw (bronze) + canonical (silver) storage to MinIO. Backfill orchestration, provider budget tracking, and multi-provider routing.

**Active providers**: EODHD (primary, 11 endpoints), Yahoo Finance (free EOD), Finnhub (free news/earnings/insider), Polygon (intraday, paid), Alpaca (1m intraday), Alpha Vantage (stub).
**Routing**: Config-driven via `routing_*` env vars (comma-separated `provider:weight` pairs). `_preferred_provider()` in `execute_task.py` selects the highest-weight registered provider.

## Authentication
- Whole app wrapped in `InternalJWTMiddleware` (PRD-0025) — every route except `/healthz`/`/readyz`
  requires a valid internal `X-Internal-JWT` bearer token (verified against S9 JWKS).
- There is NO `X-Internal-Token` static-header scheme and NO `internal_service_token` setting.
- `MARKET_INGESTION_INTERNAL_JWT_SKIP_VERIFICATION=true` bypasses verification (E2E only; rejected when APP_ENV=production).

## Provider Routing (config-backed, no DB table)
- Routing comes from `routing_*` env vars (comma-separated `provider:weight`), reloaded by restart.
  `_preferred_provider()` (in `application/use_cases/strategies/routing.py`) picks the highest-weight registered provider.
- OHLCV **EOD (1d) → Alpaca (free, IEX deep-daily)**, EODHD failover; **intraday → Alpaca, then Polygon**.
- QUOTES / FUNDAMENTALS → EODHD. NEWS_SENTIMENT / EARNINGS_CALENDAR / INSIDER_TRANSACTIONS default → EODHD
  (set the matching `routing_*` env var to `finnhub:100,eodhd:80` to prefer Finnhub's free tier).
- Yahoo Finance is DROPPED from default OHLCV routing (PLAN-0036 final topology); 1w/1mo are derived-on-read in S3, never routed here.
- Quota + circuit-breaker checks are skipped for non-EODHD providers.
- Zero-bar failover: 5 consecutive bars_returned=0 → re-fetch with fallback provider (Valkey streak counter, 24h TTL).
  `zero_bar_tracker=None` disables failover (no Valkey = no failover).

## Important Patterns

### Event IDs
- Event IDs use **UUIDv7** (migrated from ULID on 2026-04-07).
- All new IDs: `new_uuid7_str()` from `common.ids`. Never use `new_ulid()` in this service.
- Worker IDs also use UUIDv7 (WorkerProcess.__init__).

### Watermark Race
- `WatermarkViolation` → **task.retry()**, NOT task.fail().
- Two workers racing on the same watermark: the loser gets WatermarkViolation.
- The loser is re-queued via retry() so the task is attempted again by the worker loop.
- This is handled in `ExecuteTaskUseCase.execute()` — step-5 catches WatermarkViolation and calls `_persist_retry()`.

### Worker Exponential Backoff (M-008)
- `WorkerProcess._claim_batch()` uses exponential backoff on DB claim failures.
- Formula: `backoff = min(backoff * 2 + idle_sleep, 60.0)`
- With idle_sleep=5s: 5s → 15s → 35s → 60s (cap)
- Backoff resets to 0 on any successful claim.
- Logged as `backoff_seconds` field in the `worker_claim_error` log event.

### Lease-Owner Guard (N-009)
- `task_repository.save()` checks `locked_by == task.lease_owner OR locked_by IS NULL`.
- If a stale worker tries to save a task owned by a different worker, rowcount==0.
- A `task_save_lease_mismatch` WARNING is logged (no exception — self-corrects via lease expiry).

## Pitfalls
- `eodhd_base_url` is injected into `EODHDProviderAdapter` at construction — override via `MARKET_INGESTION_EODHD_BASE_URL` env var (no image rebuild needed)
- EODHD demo key only supports original 3 endpoints + limited earnings calendar
- **httpx stubs return `Any` for `response.content`** — always use `cast("bytes", response.content)` in provider adapter `_get()` methods or mypy will fail with `no-any-return`
- **Finnhub `fetch_earnings_calendar` has NO symbol param** — free tier returns all upcoming earnings in the date window; call site in `execute_task._fetch()` passes only `from_date`/`to_date`
- **Finnhub `fetch_insider_transactions` uses `ticker` not `symbol`** — must match `execute_task._fetch()` cast call at line 378: `fetch_insider_transactions(ticker=task.symbol)`
- `FinnhubProviderAdapter` is only registered when `finnhub_api_key != ""` — set `MARKET_INGESTION_FINNHUB_API_KEY` in env to activate
- **`YahooFinanceProviderAdapter` supports only daily/weekly/monthly timeframes** (`_SUPPORTED_TIMEFRAMES = {"1d","1w","1mo","1M"}`) — raises `ProviderUnavailable` for intraday (e.g. "1h", "5m"); uses `run_in_executor` for the blocking yfinance call
- **yfinance is synchronous** — NEVER call `yf.Ticker.history()` directly in async code; always wrap in `asyncio.get_running_loop().run_in_executor(None, lambda: ...)`
- **Unit-testing readyz endpoint**: override BOTH `get_uow` AND `get_read_uow` (readyz uses R27 read replica), AND set `app.state._internal_jwt_public_key = "dummy"` to pass the JWKS check (F-003B)
- Rate limit exceeded marked as `rate_limited`, not failed
- MinIO unavailable triggers RetryableError with backoff
- Negative cache (`neg:ing:{provider}:{symbol}`) prevents retry storms (120s TTL)
- Token bucket (ProviderBudget) must be loaded with `get_for_update()` inside a transaction to prevent double-consume (BP-036)
- Bronze object store uses skip-if-exists on retry path (D-008) — put() is skipped if key already exists
- Watermark must be locked with `get_for_update()` inside the step-5 transaction to prevent concurrent worker races
- `result_ref_bucket/key/sha256/mime_type/completed_at` columns added to `ingestion_tasks` — ORM and migration 0001 updated

## Multi-Provider Adapters
- `ProviderRoutingCache` (`application/services/provider_routing_cache.py`): in-memory, config-backed
  cache of `routing_*` weights; reloaded by `RoutingReloadUseCase`. There is NO `provider_routing_rules`
  DB table and NO `ProviderRoutingRule` entity — routing is purely env-var driven.
- `AlpacaProviderAdapter`: multi-symbol batch `fetch_ohlcv_batch()` via `/v2/stocks/bars`; API key in HTTP headers `APCA-API-KEY-ID` + `APCA-API-SECRET-KEY` (NEVER in URL). Primary OHLCV (EOD + intraday) source.
- `PolygonProviderAdapter`: single-ticker aggregate `/v2/aggs/ticker/{symbol}/range/...`; free tier = `asyncio.Semaphore(5)` for 5 req/min. Intraday failover.
- `IngestionTask.fetched_by_provider: str | None`: records which provider actually succeeded (ORM column added in migration 0010).

## Bug Patterns (relevant to THIS producer-only service)
- BP-036: Token bucket (ProviderBudget) requires SELECT-FOR-UPDATE inside a transaction to prevent double-consume
- BP-023: Use `uvx ruff format` (not venv ruff) for pre-commit formatting; no AM/MM Python files before committing
- BP-112: claim_batch must reclaim expired RUNNING tasks (locked_until < now) to prevent crashed-worker deadlock
- BP-113: TypeError in canonicalize (e.g., None volume field) must be caught and call task.fail() — not left RUNNING
  (BP-034/035 are Kafka-consumer dedup patterns — N/A here: this service has no consumers, only an outbox dispatcher)
