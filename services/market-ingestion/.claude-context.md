# Market Ingestion Service (S2) вЂ” Agent Context

**Port**: 8002 | **DB**: `ingestion_db` | **Status**: Production-ready (migrations 0001вЂ“0013 shipped)

## Mission
Scheduled polling of EODHD, Yahoo Finance, Finnhub, Polygon, and Alpaca. Raw (bronze) + canonical (silver) storage to MinIO. Backfill orchestration, provider budget tracking, and multi-provider routing.

**Active providers**: EODHD (primary, 11 endpoints), Yahoo Finance (free EOD), Finnhub (free news/earnings/insider), Polygon (intraday, paid), Alpaca (1m intraday), Alpha Vantage (stub).
**Routing**: Config-driven via `routing_*` env vars (comma-separated `provider:weight` pairs). `_preferred_provider()` in `execute_task.py` selects the highest-weight registered provider.

## Provider Routing (PLAN-0038 W A-4)
- `_preferred_provider()` in `execute_task.py` selects the cheapest registered provider per dataset/timeframe
- OHLCV 1d/1w/1mo/1M в†’ Yahoo Finance (free); intraday в†’ EODHD
- NEWS_SENTIMENT / EARNINGS_CALENDAR / INSIDER_TRANSACTIONS в†’ Finnhub (free tier, 60 req/min)
- FUNDAMENTALS / QUOTES / ECONOMIC_EVENTS / MACRO always в†’ EODHD
- Quota and circuit breaker checks are skipped for non-EODHD providers
- `_fallback_provider()` returns the next provider in the chain after zero-bar failover (Yahooв†’EODHD, Finnhubв†’EODHD)
- Zero-bar failover: 5 consecutive bars_returned=0 в†’ re-fetch with fallback provider (Valkey streak counter, 24h TTL)
- `zero_bar_tracker=None` disables failover (backward-compatible; no Valkey = no failover)

## Important Patterns

### Event IDs
- Event IDs use **UUIDv7** (migrated from ULID on 2026-04-07).
- All new IDs: `new_uuid7_str()` from `common.ids`. Never use `new_ulid()` in this service.
- Worker IDs also use UUIDv7 (WorkerProcess.__init__).

### Watermark Race
- `WatermarkViolation` в†’ **task.retry()**, NOT task.fail().
- Two workers racing on the same watermark: the loser gets WatermarkViolation.
- The loser is re-queued via retry() so the task is attempted again by the worker loop.
- This is handled in `ExecuteTaskUseCase.execute()` вЂ” step-5 catches WatermarkViolation and calls `_persist_retry()`.

### Worker Exponential Backoff (M-008)
- `WorkerProcess._claim_batch()` uses exponential backoff on DB claim failures.
- Formula: `backoff = min(backoff * 2 + idle_sleep, 60.0)`
- With idle_sleep=5s: 5s в†’ 15s в†’ 35s в†’ 60s (cap)
- Backoff resets to 0 on any successful claim.
- Logged as `backoff_seconds` field in the `worker_claim_error` log event.

### Lease-Owner Guard (N-009)
- `task_repository.save()` checks `locked_by == task.lease_owner OR locked_by IS NULL`.
- If a stale worker tries to save a task owned by a different worker, rowcount==0.
- A `task_save_lease_mismatch` WARNING is logged (no exception вЂ” self-corrects via lease expiry).

## Pitfalls
- `eodhd_base_url` is injected into `EODHDProviderAdapter` at construction вЂ” override via `MARKET_INGESTION_EODHD_BASE_URL` env var (no image rebuild needed)
- EODHD demo key only supports original 3 endpoints + limited earnings calendar
- **httpx stubs return `Any` for `response.content`** вЂ” always use `cast("bytes", response.content)` in provider adapter `_get()` methods or mypy will fail with `no-any-return`
- **Finnhub `fetch_earnings_calendar` has NO symbol param** вЂ” free tier returns all upcoming earnings in the date window; call site in `execute_task._fetch()` passes only `from_date`/`to_date`
- **Finnhub `fetch_insider_transactions` uses `ticker` not `symbol`** вЂ” must match `execute_task._fetch()` cast call at line 378: `fetch_insider_transactions(ticker=task.symbol)`
- `FinnhubProviderAdapter` is only registered when `finnhub_api_key != ""` вЂ” set `MARKET_INGESTION_FINNHUB_API_KEY` in env to activate
- **`YahooFinanceProviderAdapter` supports only daily/weekly/monthly timeframes** (`_SUPPORTED_TIMEFRAMES = {"1d","1w","1mo","1M"}`) вЂ” raises `ProviderUnavailable` for intraday (e.g. "1h", "5m"); uses `run_in_executor` for the blocking yfinance call
- **yfinance is synchronous** вЂ” NEVER call `yf.Ticker.history()` directly in async code; always wrap in `asyncio.get_running_loop().run_in_executor(None, lambda: ...)`
- **Unit-testing readyz endpoint**: override BOTH `get_uow` AND `get_read_uow` (readyz uses R27 read replica), AND set `app.state._internal_jwt_public_key = "dummy"` to pass the JWKS check (F-003B)
- Rate limit exceeded marked as `rate_limited`, not failed
- MinIO unavailable triggers RetryableError with backoff
- Negative cache (`neg:ing:{provider}:{symbol}`) prevents retry storms (120s TTL)
- Token bucket (ProviderBudget) must be loaded with `get_for_update()` inside a transaction to prevent double-consume (BP-036)
- Bronze object store uses skip-if-exists on retry path (D-008) вЂ” put() is skipped if key already exists
- Watermark must be locked with `get_for_update()` inside the step-5 transaction to prevent concurrent worker races
- `result_ref_bucket/key/sha256/mime_type/completed_at` columns added to `ingestion_tasks` вЂ” ORM and migration 0001 updated

## Multi-Provider Routing (PLAN-0040, pending)

- `IngestionTask.fetched_by_provider: str | None = None`: records which provider actually succeeded (populated in `task.succeed()` call site); requires ORM column + migration 0010
- `PrimaryProviderReclaimWorker`: runs every 4h; finds `fetched_by_provider != primary_for()` tasks; re-creates them for primary
- PLAN-0040 Wave A-4 supersedes PLAN-0038 Wave A-4's `_preferred_provider()` static function вЂ” `ProviderRoutingCache` replaces it when wired; `_preferred_provider()` kept as fallback when `routing_cache=None`
