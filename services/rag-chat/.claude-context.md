# RAG / Chat Service (S8) — Agent Context

**Port**: 8008 | **DB**: `rag_db` (owned, Alembic enabled) | **Status**: Waves D-1–D-4, E-1–E-3, F-1–F-4, G-1 done — full 13-step RAG pipeline + SSE streaming + chat endpoints + S9 gateway routes + per-source circuit breaker (PLAN-0031 D-1)

## Mission
Hybrid intelligence RAG pipeline: entity resolution → parallel retrieval (vector ANN + KG traversal + SQL structured) → fusion + reranking → LLM completion with SSE streaming + contradiction flagging. Conversation persistence in `rag_db`.

## Key Invariants
- `fusion_score = score × recency_score × trust_weight` — deviation > 1e-9 is a programming error
- `recency_score = exp(-0.005 × days_old)` where `days_old = (now() - published_at).days`; `published_at is None` → `recency_score = 0.5`
- Trust weights: `sec_10k/10q → 0.95`, `earnings_data → 0.95`, `eodhd_news → 0.70`, `finnhub_news → 0.65`, `other → 0.50`
- Input max length: 2000 chars after HTML strip; empty after strip → EMPTY error

## Pipeline (13 steps)
1. Input validation (bleach strip → truncate → PII regex → injection heuristic → XML-wrap output)
2. Intent classification (Qwen2.5:3b; returns `QueryIntent` + entity hints)
3. Entity resolution (aliases lookup via S6 ANN endpoint or local entity search)
4. Query decomposition (only for COMPARISON/REASONING with ≥2 entities)
5. HyDE generation (Qwen2.5:7b-instruct; generates hypothetical answer paragraph, embeds via BGE)
6. Parallel retrieval (`asyncio.gather`):
   - ANN search (S6 `/api/v1/vector-search` with HyDE embedding, top-30)
   - KG traversal (S7 `/api/v1/entities/{id}/graph`, 2-hop)
   - SQL structured (S3 fundamentals/OHLCV endpoints, FINANCIAL_DATA/PORTFOLIO intents only)
7. Result fusion (normalize scores → compute fusion_score → deduplicate by source_id)
8. BGE reranking (cross-encoder `bge-reranker-v2-m3` via Ollama, top-30 → top-12)
9. Context assembly (inline KG relations, inject `⚠️ CONFLICTING EVIDENCE` blocks for contradictions)
10. LLM completion stream (Qwen2.5:7b-instruct via provider chain; SSE `token` events)
11. Citation extraction (parse `[N]` markers from output → Citation objects)
12. Output processing (strip `<think>`/`<reasoning>` tags, format citation list)
13. Persistence (save thread + message to `rag_db`)

## LLM Provider Fallback Chain
DeepInfra (primary, if `RAG_CHAT_DEEPINFRA_API_KEY` set) → OpenRouter (if `RAG_CHAT_OPENROUTER_API_KEY` set) → Ollama (emergency fallback, always present)
Negative cache: `rag:v1:neg:{provider_name}` in Valkey, TTL 60s. A failed provider is skipped for 60s.
All providers unavailable → `ProviderUnavailableError` → 503 to client.

**Classification model**: `qwen2.5:3b` (fast, local Ollama)
**Completion model (primary)**: DeepSeek R1 Distill 32B via DeepInfra; fallback: OpenRouter → Ollama (`deepseek-r1:32b`)
**Embedding model**: BGE 1024-dim via S6 `/api/v1/embed` endpoint (not libs/ml-clients directly)
**Reranker model**: `bge-reranker-v2-m3` via Ollama cross-encoder endpoint

## Process Topology (R22 — PLAN-0015-D Wave D-3)
S8 lifespan is clean — NO `asyncio.create_task()`.
Single process: FastAPI ASGI server only. No background workers or Kafka consumers.

## Critical Rules
- **R25**: API routes only import use case classes — never `infrastructure/` directly
- **R27**: Threads/messages reads use `ReadOnlyUnitOfWork`; writes use `UnitOfWork`
- **R26**: No auto-commit in `UnitOfWork.__aexit__` — explicit `await uow.commit()` in every use case
- **R22**: No `asyncio.create_task()` in app lifespan — ASGI only
- **Never** store raw LLM output in DB — always strip `<think>` / `<reasoning>` blocks first
- **Never** interpolate user input into prompt templates without XML-wrapping
- Rate limit enforced per `(tenant_id, user_id)` via Valkey sliding window
- All upstream service calls (S6, S7, S3, S1) use `httpx.AsyncClient` with timeouts
- Thread ownership: every thread read/write verifies `thread.tenant_id == request_tenant_id`

## Pitfalls
- `bleach` required as a dependency for HTML stripping in `InputValidator` — must be in pyproject.toml
- All `valkey` constructor parameters must be typed as `ValkeyClient` (from `messaging.valkey.client`), never `aioredis.Redis` or `redis.asyncio.Redis` — see BP-103b. New `ValkeyClient` methods must be staged in the same commit as the callers to avoid pre-commit hook `attr-defined` failures.
- `asyncio.gather` in parallel retrieval: use `return_exceptions=True` to prevent one failing source from aborting all retrieval
- SSE stream: if client disconnects, `asyncio.CancelledError` is raised — must clean up any open `httpx` connections
- `fusion_score` must be computed atomically — do not pass partial `RetrievedItem` without all 3 factor fields set
- Negative provider cache key format: `rag:v1:neg:{provider_name}` (not just `rag:neg:`)
- BGE reranker input format: `(query, passage)` pairs — NOT `(passage, query)` (model is asymmetric)
- `thread.tenant_id` ownership check MUST happen in the use case, not just in the route — defense in depth
- Contradiction detection: only inject warning block when S7 returns `is_contradicted=true` on a relation
- CYPHER_ENABLED=false is the default — Phase 4 Cypher endpoint not implemented in PLAN-0015

## Context Manager Notes (Wave A-3)
- TTL constants (`_CHUNK_TTL=4h`, `_SUMMARY_TTL=24h`) are defined in `context_manager.py` (application layer) — NOT imported from `valkey_chunk_cache.py`. Architecture boundary: application must not import from infrastructure at runtime.
- `_cosine_similarity` / `_entity_overlap` are pure Python helpers in `context_manager.py` — no external dependencies.
- `ContextManager._background_tasks: set[asyncio.Task]` holds strong references to fire-and-forget summary tasks; tasks clean themselves up via `done_callback(discard)`.
- `_deserialize_chunk` recomputes `fusion_score = score * recency_score * trust_weight` from stored components to avoid float JSON round-trip drift.
- Entity overlap: when BOTH turns have no entities, `_entity_overlap` returns 1.0 (vacuously satisfied) so GENERAL intent queries can still benefit from chunk cache reuse.
