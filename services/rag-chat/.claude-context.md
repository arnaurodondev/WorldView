# RAG / Chat Service (S8) — Agent Context

**Port**: 8008 | **DB**: None (stateless) | **Status**: Compliant

## Mission
Query rewriting, intent classification, hybrid retrieval (vector + KG + SQL), result fusion, reranking, context assembly, LLM completion, citation injection, response caching.

## Domain Entities
ChatRequest, ChatResponse, Citation

## Kafka Topics
None — stateless orchestrator.

## LLM Provider Fallback Chain
Ollama (10s) → Groq (5s) → OpenRouter (10s) → OpenAI (10s)
Negative cache (60s) prevents retry storms on failing providers.

## API Endpoints (4)
healthz, readyz, chat (sync), stream (SSE), provider status

## Critical Rules
- Rate limit: 10 queries/min per tenant
- Token budget: 4000 output tokens, 8000 context tokens max
- Output sanitization: strip `<think>` / `<reasoning>` blocks from LLM output
- In-flight dedup prevents thundering herd on repeated queries

## Pitfalls
- Caching: query rewrite (1h), completion (24h), provider negative (60s)
- No database — all state is transient or cached in Valkey
- Must sanitize LLM output before sending to client (prompt injection defense)
- User input must be sanitized before prompt construction

## Security Concerns
- **Prompt injection**: User queries interact with LLMs — sanitize input
- **Data leakage**: Retrieval results must be filtered by tenant
- **LLM output safety**: Validate and sanitize before presentation
