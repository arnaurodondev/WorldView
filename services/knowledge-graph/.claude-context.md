# Knowledge Graph Service (S7) — Agent Context

**Port**: 8007 | **DB**: `intelligence_db` (read/write, DDL NOT owned) | **Status**: Compliant

## Mission
Relation canonicalization, graph materialization + evidence staging, async workers (aggregation, confidence decay, contradiction detection, summary generation, embedding refresh). Shadow migration to Apache AGE planned.

## Domain Entities
Relation, RelationType, Contradiction

## Kafka Topics
| Direction | Topic | Notes |
|-----------|-------|-------|
| Produces | `graph.state.changed.v1` | Graph state updates |
| Produces | `intelligence.contradiction.v1` | Contradictions detected |
| Produces | `relation.type.proposed.v1` | New relation types proposed |
| Produces | `entity.dirtied.v1` | Compacted topic — latest state per entity |
| Consumes | `nlp.article.enriched.v1` | Enriched articles from S6 |

## Database
**intelligence_db** (DDL owned by intelligence-migrations, S7 read/write only):
canonical_entities, relation_type_registry, relations (hash-partitioned 8x on subject_entity_id), relation_evidence_raw, relation_evidence, relation_summaries, article_claims, contradictions

## Async Workers
- Aggregation (300s cycle)
- Contradiction detection (30s)
- Summary generation (3600s)

## Critical Rules
- **NEVER** run Alembic against `intelligence_db` from this service (`ALEMBIC_ENABLED=false`)
- Two semantic modes: RELATION_STATE (active/inactive, event-triggered invalidation) vs TEMPORAL_CLAIM (historical, usually not invalidated)
- `entity.dirtied.v1` is a compacted topic — key = entity_id

## API Endpoints (4)
graph neighborhood, relations query, graph stats

## Pitfalls
- Relations table hash-partitioned on `subject_entity_id` (8 partitions)
- Apache AGE available but NOT used in v1
- Confidence formula is 4-step, bounded [0,1]
