## Mission
Relation canonicalization (Block 11), graph materialization + evidence staging (Block 12 hot path), 15 async APScheduler workers (Block 13 — confidence recomputation, contradiction detection, summary generation, entity descriptions, embedding refresh, partition management, narrative generation, path pre-computation), Apache AGE shadow sync (Block 14), outbox dispatcher.

## Domain Entities
- `Relation` (frozen DC) — maps to hash-partitioned `relations` table (HASH ×8 on `subject_entity_id`)
- `RelationEvidence` (frozen DC) — `is_backfill` flag for historical loads
- `RelationSummary` — LLM-generated; `evidence_hash` for change-detection skip
- `ContradictionLink` — row in `relation_contradiction_links`
- `Contradiction` — event aggregate: subject-based, opposite+non-neutral polarity, 90-day window
- `ConfidenceComponents` — 4-step bounded formula result; call `.validate()` after construction
- `SemanticMode` StrEnum: `RELATION_STATE | TEMPORAL_CLAIM`
- `DecayClass` StrEnum: `STANDARD | TEMPORAL`
- `RelationType` — 16 code-level values; more total in `relation_type_registry` DB seeds
- `TemporalEvent` (frozen DC) — `lifecycle_phase` computed at access time (PENDING_ACTIVE/ACTIVE/RESIDUAL/EXPIRED)
- `EntityEventExposure` (frozen DC) — entity ↔ temporal event link
- `SimilarEntityResult` (frozen DC, slots=True) — ANN similarity result; gains `surprise_score: float | None`
- `EntityNarrativeVersion` — version-controlled LLM narrative with `health_score`

## Confidence Formula
4-step bounded: support (temporal-weighted avg of source_weights) → corroboration gain (distinct qualifying sources, cap 0.20) → contradiction penalty (top-3 decayed links, cap 0.60) → clamp [0,1].
- RELATION_STATE: uses `decay_alpha` from `decay_class_config` row
- TEMPORAL_CLAIM: always uses `0.02310` (30-day half-life)

## Database
**intelligence_db** (DDL owned by intelligence-migrations, `ALEMBIC_ENABLED=false`):
- **Dual sessions** (R23 + R27): `create_intelligence_session_factory` (write) + `create_readonly_session_factory` (read-only)
- **Critical invariants**:
  - `partition_key` in `relations` and `relation_evidence_raw` is `GENERATED ALWAYS AS STORED` — **NEVER include in INSERT**
  - `relation_evidence_raw` has NO `relation_id` column — JOIN on the triple to get evidence for a relation
  - `relation_evidence_raw` has NO `canonicalized_evidence_text` column — that column is only on `relation_evidence` (partitioned processed table)
  - `relation_evidence_raw.processed` (ConfidenceWorker 13A = "confidence recomputed") and `relation_evidence_raw.promoted_at` (RelationEvidencePromoterWorker 13B = "promoted to relation_evidence", added in migration 0061) are DISTINCT markers — the promoter filters `promoted_at IS NULL` so it never re-scans the already-promoted backlog
  - `entity.dirtied.v1` must be produced AFTER `session.commit()`, never before
  - `temporal_events.event_type` CHECK accepts `prediction` as of migration 0066 (PLAN-0056 Sub-Plan C KEYSTONE); `EventType.PREDICTION` + the `PredictionEnrichedConsumer` (Wave C2) now write these rows. The consumer runs its OWN group (`kg-prediction-enriched-group`) on `nlp.article.enriched.v1`, filters `source_type == 'polymarket'` (the real value B2 emits — NOT `'prediction_market'`), and per Polymarket synthetic doc upserts 1 `temporal_events(prediction, scope=LOCAL, region='prediction', title=f"Prediction market {doc_id}")` + 1 `entity_event_exposures(directly_affected)` per resolved entity. The enriched event has NO title/condition_id (only doc_id/source_type/resolved_entity_ids/published_at/occurred_at), so `doc_id` anchors the idempotent natural key; recovering the real question/condition_id would need an R7 nlp_db read (forbidden)
  - `entity_event_exposures` gained `polarity` (`bullish`/`bearish`/`neutral`, VARCHAR(20)+`ck_exposure_polarity` CHECK, nullable — NOT a PG enum per BP-007) and `polarity_confidence` (double, nullable) in migration 0066 — populated only for prediction-market exposures; existing earnings/corporate exposures keep NULL polarity

## Key Pitfalls
- **LLM cost is resolved via `ml_clients.pricing.resolve_cost` (PLAN-0117 W3, R13)** — `FallbackChainClient` no longer uses local char-count cost heuristics. Each `_log(...)` call resolves `(Decimal, cost_source)`: DeepInfra extraction prefers `ExtractionOutput.provider_cost_usd` (surfaced by the deepseek adapter, FR-1) → `"provider"`; Gemini → `"pricematrix"`; Ollama → `"local"` ($0). `cost_source`/`user_id` (NULL for KG background pipelines) are threaded through `SessionScopedKgUsageLogger` → `LlmUsageLogRepository` raw-SQL INSERT into `intelligence_db.llm_usage_log` (cols added by intel-migrations `0064`, W2). Never hardcode `estimated_cost_usd=0.0` for a paid model.
- **Silent-zero cost guard (PLAN-0117 W5, FR-7, BP-715)** — the `LlmUsageLogRepository.log` INSERT now calls `observability.metrics.record_silent_zero_cost("knowledge-graph", …)`, tripping `llm_usage_silent_zero_cost_total{service,model_id}` for a tokens>0 / `$0` / paid-source row (`cost_source NOT IN local, aggregate`). The scheduler boot logs a WARNING for any configured model with no pricing path (`warn_unpriceable_models`); new configured model ids must be added to `ml_clients.model_registry.PLATFORM_MODEL_REGISTRY` (CI-enforced).
- **`partition_key` STORED** — never in INSERT; DB raises error
- **`relation_evidence_raw` has NO `relation_id`** — JOIN on the triple
- **`relation_evidence_raw` has NO `canonicalized_evidence_text`** — only `relation_evidence` (processed table) has it
- **AGE Cypher injection** — entity IDs as `$params`; edge labels only exception (validate against `_VALID_EDGE_LABELS` whitelist)
- **AGE `LOAD 'age'` per connection** — handled by AGE adapter; do NOT use read-only sessions for Cypher
- **AGE O(n³) at depth=3 on hub entities** — 5s timeout enforced; use pre-computed `path_insights` for scale
- **`entity.dirtied.v1` post-commit ordering** — accumulate `entity_ids_to_dirty`, produce AFTER commit
- **`ensure_rows_exist()` entity type check** — `financial_instrument` → 3 rows; all others → 2 rows (`definition` + `narrative` only)
- **Ollama model dimension** — MUST use `bge-large:latest` (1024-dim); `nomic-embed-text` (768-dim) raises FatalError
- **GLOBAL temporal events** — link to sector/industry entities ONLY; never per-company rows
- **EODHD API fields that do NOT exist**: `General.Officers`, `Holders.Institutions`, `Financials.Revenue_Segment` — use Insider Transactions API instead
- **`create_or_get()` partial index BP-459** — unique index on `lower(canonical_name)` has `WHERE entity_type != 'financial_instrument'`; phantom duplicates possible for provisional entities with name variations
- **`EmbeddingClientProtocol` is duck-typed locally** — no ml-clients runtime dep in knowledge-graph; Python version boundary
- **`SummaryWorker` ARCH-003 3-phase** — Phase 1+2 on read replica, Phase 3 (LLM) holds no session, Phase 4 opens fresh write session per row
- **`summary_worker_force_regen_batch_size=0` default** — only set to positive value for one tick after prompt template upgrades
- **`_NOISE_BLOCKLIST` location** — `infrastructure/workers/provisional_enrichment.py` module-level frozenset
- **`provisional_entity_queue.status` has 5 values**: `pending/processing/resolved/failed/noise` (CHECK constraint migration 0020)
- **PLAN-0112 weird-path engine (`AgeGraphPathEngine`)**:
  - **VLE must be UNTYPED** (`-[*L..L]-`), NOT multi-label (`-[:A|B*L..L]-` is a hard AGE-1.5 parse error at `|`, BP-689). Membership pruning is a **post-hoc Python filter** (drop paths whose `rel_types` ∩ `MEMBERSHIP_RELATIONS`), NOT a typed allow-list on the pattern. NEVER emit the explicit untyped-edge form `-[r]-` (18.4 s seq-scan vs 0.24 s VLE).
  - **Session GUCs are `SET`, not `SET LOCAL`** — `SET LOCAL` evaporated before the traversal transaction (the original Postgres-flood bug). `SET statement_timeout` + `SET max_parallel_workers_per_gather=0` on the same session as the query.
  - **`path_max_hops` capped at 3** — post-hoc filter prunes results not the frontier, so hop-4/5 blow up.
  - **Novelty first-seen** = `COALESCE(relations.first_evidence_at, MIN(relation_evidence.evidence_date))` — bridges the AGE↔relations sync gap (FR-13); without the COALESCE novelty is uniformly 0.
  - **`dst_entity_id` FK is NULL-guarded** — non-canonical/duplicate endpoints would violate the FK; prefetch + null-guard before persist.
  - **Pairwise `connected`/`shortest_hops` derive from the SAME distinct-node-filtered path set** the endpoint returns (not a separate looser existence probe) — when the only path routes through a duplicate-canonical vertex it is reported not-connected until FR-11 dedup lands.
  - `node_degree`/`graph_stats` refreshed each AGE-sync cycle via a fast `_ag_label_edge` SQL agg (Cypher degree-enum was 50 s → timeout; use the raw table).
- **Relational traversal engine (`RelationalGraphPathAdapter`, `infrastructure/relational/graph_path_adapter.py`)** — a 2nd `GraphPathEngine` impl over the `public.graph_edges` materialized view (migration 0063), selected at DI when `KNOWLEDGE_GRAPH_RELATIONAL_TRAVERSAL_ENABLED=true` (default false → AGE). No AGE / no `LOAD 'age'` / no write session → runs on `read_factory` (R27-clean).
  - `public.graph_edges` = both-directions, dedup projection of `relations` (`confidence>0.1`, no self-loops; `typ=UPPER(REPLACE(canonical_type,' ','_'))`; carries `subject_entity_id` so `edge_forward[i] = (node_path[i] == subject_entity_id)` matches AGE directionality). MUST be **schema-qualified `public.graph_edges`** — a stray identical matview lives in `ag_catalog` on some AGE pg images and an unqualified ref resolves to it under the AGE search_path. Refresh after big ingests: `REFRESH MATERIALIZED VIEW CONCURRENTLY public.graph_edges` (unique idx `uidx_graph_edges_rel_src_dst` supports it).
  - `path_exists` = settled-set recursive CTE (`UNION`, dedup) → shortest hop; `find_paths_between`/`find_paths_from_anchor` = path-array CTE (`UNION ALL`) carrying `node_path`/`rel_path`, in-SQL cycle guard (`NOT dst = ANY(node_path)`), per-node degree cap (`KNOWLEDGE_GRAPH_RELATIONAL_TRAVERSAL_DEGREE_CAP`, default 200). Reuses `MEMBERSHIP_RELATIONS` post-hoc prune; rejects self-loops; maps statement-timeout → `CypherTimeoutError` (HTTP 503), same as AGE.
  - Live (idle) bench vs AGE on identical mid-degree pairs: relational connectivity p50 25 ms / p95 41 ms vs AGE staged p50 61 ms / p95 180 ms; relational has no AGE-write-session contention (AGE degrades to ~17 s under load). `MEMBERSHIP_RELATIONS` grew to 6 (added `IS_IN_INDUSTRY`, `REVENUE_FROM_COUNTRY`).
