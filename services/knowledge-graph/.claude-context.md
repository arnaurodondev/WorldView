## Confidence Formula
4-step bounded: support (temporal-weighted avg of source_weights) → corroboration gain (distinct sources, cap 0.20) → contradiction penalty (top-3 decayed links, cap 0.60) → clamp to [0,1].
- RELATION_STATE: uses decay_alpha from decay_class_config
- TEMPORAL_CLAIM: always uses 0.02310 (30-day half-life)

## Critical Rules
- **NEVER** run Alembic against `intelligence_db` (`ALEMBIC_ENABLED=false` → RuntimeError guard in session factory)
- `partition_key` in `relations` and `relation_evidence_raw` is `GENERATED ALWAYS AS STORED` — **NEVER include in INSERT**
- `entity.dirtied.v1` is a compacted topic — direct produce (bypasses outbox), key = entity_id
- Advisory lock (`pg_advisory_xact_lock`) on triple hash before relation upsert
- Contradiction requires: same subject, opposite polarity, both non-neutral, within 90-day window

## Pitfalls
- Relations table hash-partitioned ×8 on `subject_entity_id`; upsert needs advisory lock
- `partition_key` STORED column — omit from all INSERTs
- Single evidence piece: decay cancels in support formula (support = source_weight regardless of alpha)
- To test alpha differentiation, use ≥2 evidence pieces at different ages
- `make_interval(days => :param)` for parameterized INTERVAL in PostgreSQL (avoids f-string injection)
- `readyz` returns 503 when DB is not configured (test client); integration tests needed for 200 path
- DLQ admin auth: empty `KNOWLEDGE_GRAPH_ADMIN_TOKEN` disables all DLQ access (not enables it)
- `list_for_entity` / `list_filtered` use static parameterized SQL with `IS NULL OR ...` pattern — no f-strings
- **`ensure_rows_exist()` entity type check** *(PLAN-0017)*: `financial_instrument` → 3 rows; all other entity types → 2 rows (`definition` + `narrative` only). Never call with hardcoded `ALL_VIEW_TYPES` for non-company entities.
- **Cypher injection prevention** *(PLAN-0018)*: AGE Cypher queries MUST use `$parameterized` syntax. Never f-string `entity_id` into Cypher. Use `ag_catalog.cypher()` with params dict.
- **AGE edge labels** *(PLAN-0018)*: Edge label must be an uppercase string matching the relation's `canonical_type` (e.g., `COMPETES_WITH`). Dynamic label selection via Python string — validated against whitelist before use.
- **GLOBAL temporal events** *(PLAN-0018)*: GLOBAL scope events link to sector/industry entities ONLY in `entity_event_exposures`. Creating per-company rows for GLOBAL events causes table explosion. Company exposure is inferred at query time via `is_in_sector` traversal.
- **EODHD section availability** *(PLAN-0018)*: `General.Officers`, `Holders.Institutions`, `Financials.Revenue_Segment` are **confirmed non-existent in the EODHD API** (not a tier issue — see ADR-0018-005). Do not attempt to read these fields. Use the replacement APIs: Insider Transactions API for executives, SharesStats.PercentInsiders/PercentInstitutions for ownership signals.
- **`entity_embedding_state` COALESCE upsert** (PLAN-0017 A-1): `ON CONFLICT DO UPDATE SET embedding = COALESCE(EXCLUDED.embedding, entity_embedding_state.embedding)`. Never overwrite a valid vector with NULL when only the `source_hash` changed.
- **`DefinitionRefreshWorker` 3-phase R24 compliance** (PLAN-0017 A-1): Phase 1 = read due rows (session closed), Phase 2 = generate descriptions + embed (no session open), Phase 3 = upsert results (new session). DB session MUST NOT be held across Gemini HTTP calls.
- **`InstrumentEntityConsumer` BP-124 fix** (PLAN-0017 A-1): Idempotency checks `model_id IS NULL` in `entity_embedding_state`. If entity exists but embedding is absent (model_id=None), still calls `refresh_for_entity` to recover from mid-flight crash.
- **`get_batch()` on CanonicalEntityRepository** (PLAN-0017 A-1): Uses single `WHERE entity_id = ANY(:ids)` query. Pass `[str(eid) for eid in entity_ids]` — not raw UUIDs. Missing IDs are silently omitted (no error).
- **FundamentalsRefreshWorker earnings dedup** (Wave C-4): SELECT before INSERT on `(entity_id, event_type, quarter, fiscal_year)` from `structured_data` JSONB; `ON CONFLICT DO NOTHING` as DB-level fallback. `events.doc_id` is NOT NULL — use `new_uuid7()` as synthetic ID for earnings events.
- **FundamentalsRefreshWorker sector relations** (Wave C-4): Uses `CanonicalEntityRepository.find_by_name_and_type()` to look up seeded GICS entities (migration 003). `relation_evidence_raw.source_document_id` is NOT NULL — use `new_uuid7()` as synthetic ID. `partition_key` STORED — omit from INSERT as always.
- **HTTP client sharing** (Wave C-4): `FundamentalsRefreshWorker.run()` creates one `httpx.AsyncClient` per run, shared across `_insert_earnings_events`, `_upsert_sector_relations`, `_build_fundamentals_narrative`. `aclose()` in `finally`.
