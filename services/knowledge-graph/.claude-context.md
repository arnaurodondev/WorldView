## Mission
Relation canonicalization (Block 11), graph materialization + evidence staging (Block 12 hot path), 8 async APScheduler workers (Block 13 ‚Äî confidence recomputation, contradiction detection, summary generation, embedding refresh, partition management), outbox dispatcher.

## Domain Entities (Wave D-1)
- `Relation`, `RelationEvidence` (frozen DC, `is_backfill`), `RelationSummary`, `ContradictionLink`, `Contradiction`
- `ConfidenceComponents` ‚Äî call `.validate()` after construction
- `SemanticMode` (RELATION_STATE | TEMPORAL_CLAIM), `DecayClass` (STANDARD | TEMPORAL), `RelationType` (8 values)

## Confidence Formula
4-step bounded: support (temporal-weighted avg of source_weights) ‚Üí corroboration gain (distinct sources, cap 0.20) ‚Üí contradiction penalty (top-3 decayed links, cap 0.60) ‚Üí clamp to [0,1].
- RELATION_STATE: uses decay_alpha from decay_class_config
- TEMPORAL_CLAIM: always uses 0.02310 (30-day half-life)

## Database

## Critical Rules
- **NEVER** run Alembic against `intelligence_db` (`ALEMBIC_ENABLED=false` ‚Üí RuntimeError guard in session factory)
- `partition_key` in `relations` and `relation_evidence_raw` is `GENERATED ALWAYS AS STORED` ‚Äî **NEVER include in INSERT**
- `entity.dirtied.v1` is a compacted topic ‚Äî direct produce (bypasses outbox), key = entity_id
- Advisory lock (`pg_advisory_xact_lock`) on triple hash before relation upsert
- Contradiction requires: same subject, opposite polarity, both non-neutral, within 90-day window

## Pitfalls
- Relations table hash-partitioned √ó8 on `subject_entity_id`; upsert needs advisory lock
- `partition_key` STORED column ‚Äî omit from all INSERTs
- Single evidence piece: decay cancels in support formula (support = source_weight regardless of alpha)
- To test alpha differentiation, use ‚â•2 evidence pieces at different ages
- `make_interval(days => :param)` for parameterized INTERVAL in PostgreSQL (avoids f-string injection)
- `readyz` returns 503 when DB is not configured (test client); integration tests needed for 200 path
- DLQ admin auth: empty `KNOWLEDGE_GRAPH_ADMIN_TOKEN` disables all DLQ access (not enables it)
- `list_for_entity` / `list_filtered` use static parameterized SQL with `IS NULL OR ...` pattern ‚Äî no f-strings
