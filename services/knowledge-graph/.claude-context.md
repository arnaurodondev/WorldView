## Confidence Formula
4-step bounded: support (temporal-weighted avg of source_weights) → corroboration gain (distinct sources, cap 0.20) → contradiction penalty (top-3 decayed links, cap 0.60) → clamp to [0,1].
- RELATION_STATE: uses decay_alpha from decay_class_config
- TEMPORAL_CLAIM: always uses 0.02310 (30-day half-life)

## Critical Rules
- **NEVER** run Alembic against `intelligence_db` (`ALEMBIC_ENABLED=false` → RuntimeError guard in session factory)
- `partition_key` in `relations` and `relation_evidence_raw` is `GENERATED ALWAYS AS STORED` — **NEVER include in INSERT**
- `entity.dirtied.v1` is a compacted topic — direct produce (bypasses outbox), key = entity_id
- Advisory lock (`pg_advisory_xact_lock`) on triple hash before relation upsert
- Contradiction requires: same subject, opposite polarity, both non-neutral, within 90-day window
- **`relation_evidence_raw` has NO `relation_id` column** — it stores the triple (subject_entity_id + object_entity_id + canonical_type). To get evidence for a relation, JOIN to `relations` on the triple. Never assume `WHERE relation_id = ANY(:ids)` works on `relation_evidence_raw`. (PLAN-0072 Wave 2 CTE pattern: rank via `ROW_NUMBER() OVER (PARTITION BY r.relation_id)` after the JOIN.)
- **`relation_evidence_raw` has NO `canonicalized_evidence_text` column** — only `evidence_text`. `canonicalized_evidence_text` exists only on the immutable `relation_evidence` table (monthly partition). SummaryWorker reads `canonicalized_evidence_text` from `relation_evidence` via `get_all_for_relation`, not from raw.

- `list_for_entity` / `list_filtered` use static parameterized SQL with `IS NULL OR ...` pattern — no f-strings
- **`ensure_rows_exist()` entity type check** *(PLAN-0017)*: `financial_instrument` → 3 rows; all other entity types → 2 rows (`definition` + `narrative` only). Never call with hardcoded `ALL_VIEW_TYPES` for non-company entities.
- **`create_or_get()` partial index phantom duplicates** *(BP-459)*: The unique index on `canonical_entities(lower(canonical_name))` has a `WHERE entity_type != 'financial_instrument'` predicate — financial_instrument rows are excluded from conflict detection. A provisional entity with a name variation (e.g., "Amazon Business") will insert as a NEW row even when "Amazon Inc." already exists as `financial_instrument`. Fix requires removing the partial index predicate AND adding a `class_aware_canonical_match()` pre-insert lookup in `provisional_enrichment_core.py` before calling `create_or_get()`.
- **Provisional enrichment has NO pre-insert dedup** *(BP-459)*: `ProvisionalEnrichmentWorker` calls the LLM to get a canonical name, then immediately calls `create_or_get()`. There is no lookup against existing entities before insertion. The fuzzy `class_aware_canonical_match()` in `entity_alias.py` (added for S8 RAG in PLAN-0087) must be wired into this path.
- **Cypher injection prevention** *(PLAN-0018)*: AGE Cypher queries MUST use `$parameterized` syntax. Never f-string `entity_id` into Cypher. Use `ag_catalog.cypher()` with params dict. Edge labels are the ONLY exception — they cannot be parameterized in Cypher; validate against `_VALID_EDGE_LABELS` whitelist and use `_build_relation_merge_sql(edge_label)` pattern.
- **AGE edge labels** *(PLAN-0018)*: Edge label must be an uppercase string matching the relation's `canonical_type` (e.g., `COMPETES_WITH`). Dynamic label selection via Python string — validated against whitelist before use.
- **GLOBAL temporal events** *(PLAN-0018)*: GLOBAL scope events link to sector/industry entities ONLY in `entity_event_exposures`. Creating per-company rows for GLOBAL events causes table explosion. Company exposure is inferred at query time via `is_in_sector` traversal.
- **EODHD section availability** *(PLAN-0018)*: `General.Officers`, `Holders.Institutions`, `Financials.Revenue_Segment` are **confirmed non-existent in the EODHD API** (not a tier issue — see ADR-0018-005). Do not attempt to read these fields. Use the replacement APIs: Insider Transactions API for executives, SharesStats.PercentInsiders/PercentInstitutions for ownership signals.
- **`entity_embedding_state` COALESCE upsert** (PLAN-0017 A-1): `ON CONFLICT DO UPDATE SET embedding = COALESCE(EXCLUDED.embedding, entity_embedding_state.embedding)`. Never overwrite a valid vector with NULL when only the `source_hash` changed.
- **R25 — API routes must never import from `infrastructure/`** (IG-LAYER-002): All repository instantiation belongs in factory functions in `api/dependencies.py` using `Depends()`. Each repo port gets a `get_<name>_repo(session: ReadOnlyDbSessionDep)` factory + `Annotated[Port, Depends(factory)]` type alias. Routes receive the port-typed dep directly — no `from ... import Repo` inside route bodies. Pattern already used by `get_dlq_use_case`, `get_entity_graph_repos`, `get_cypher_bundle`.
- **`entity.dirtied.v1` post-commit ordering** (PLAN-0031 C-1): `materialize_graph()` returns `entity_ids_to_dirty: frozenset[UUID]` — the caller (`EnrichedArticleConsumer.process_message`) produces Kafka messages ONLY AFTER `await session.commit()`. Never produce before commit — orphaned Kafka messages will appear for rolled-back entities. Workers follow the same pattern: accumulate `entity_ids_to_dirty`, produce post-commit.
- **`DefinitionRefreshWorker` 3-phase R24 compliance** (PLAN-0017 A-1): Phase 1 = read due rows (session closed), Phase 2 = generate descriptions + embed (no session open), Phase 3 = upsert results (new session). DB session MUST NOT be held across Gemini HTTP calls.
- **`InstrumentEntityConsumer` BP-124 fix** (PLAN-0017 A-1): Idempotency checks `model_id IS NULL` in `entity_embedding_state`. If entity exists but embedding is absent (model_id=None), still calls `refresh_for_entity` to recover from mid-flight crash.
- **`get_batch()` on CanonicalEntityRepository** (PLAN-0017 A-1): Uses single `WHERE entity_id = ANY(:ids)` query. Pass `[str(eid) for eid in entity_ids]` — not raw UUIDs. Missing IDs are silently omitted (no error).
- **FundamentalsRefreshWorker earnings dedup** (Wave C-4): SELECT before INSERT on `(entity_id, event_type, quarter, fiscal_year)` from `structured_data` JSONB; `ON CONFLICT DO NOTHING` as DB-level fallback. `events.doc_id` is NOT NULL — use `new_uuid7()` as synthetic ID for earnings events.
- **FundamentalsRefreshWorker sector relations** (Wave C-4): Uses `CanonicalEntityRepository.find_by_name_and_type()` to look up seeded GICS entities (migration 003). `relation_evidence_raw.source_document_id` is NOT NULL — use `new_uuid7()` as synthetic ID. `partition_key` STORED — omit from INSERT as always.
- **HTTP client sharing** (Wave C-4): `FundamentalsRefreshWorker.run()` creates one `httpx.AsyncClient` per run, shared across `_insert_earnings_events`, `_upsert_sector_relations`, `_build_fundamentals_narrative`. `aclose()` in `finally`.
- **leidenalg lazy import** *(PRD-0023)*: `import leidenalg` and `import igraph` MUST be inside `CommunityDetectionWorker.run()`, not at module level. The API server process must never import leidenalg. Use `TYPE_CHECKING` guards in type annotations.
- **community_id is UUIDv5, not UUIDv7** *(PRD-0023)*: `community_id = uuid.uuid5(uuid.NAMESPACE_DNS, str(anchor_entity_id))`. This is a deliberate exception to R10 for stability. Do not generate UUIDv7 for community keys.
- **entity_communities UNIQUE constraint is partial** *(PRD-0023)*: `UNIQUE (entity_id) WHERE removed_at IS NULL`. Active membership is one row per entity; historical rows accumulate without constraint.
- **GraphEvolutionWorker watermark singleton** *(PRD-0023)*: `graph_evolution_watermarks` has exactly one row (id=1). Seed it with `INSERT ... ON CONFLICT DO NOTHING` at worker startup. Missing row = first run with epoch watermark (intentional — processes all history on first run).
- **CommunityDetectionWorker circuit breaker** *(PRD-0023)*: If `entity_count > 500_000`, log warning and skip the Leiden run. Prevents OOM in degenerate cases. Threshold is hardcoded `MAX_COMMUNITY_ENTITIES = 500_000`.
- **surprise_score null before first community run** *(PRD-0023)*: `SimilarEntityResult.surprise_score` is `None` until `CommunityDetectionWorker` runs at least once. API callers must handle nullable field.
