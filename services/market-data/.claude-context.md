# Market Data Service (S3) — Agent Context

**Port**: 8003 | **DB**: `market_data_db` (TimescaleDB) | **Status**: Mature

## Mission
Materialize OHLCV bars, quotes, and fundamentals from claim-check pointers. Serve query APIs for charts and financial data. Manage instrument lifecycle events.

## Domain Entities
Security, Instrument, OhlcvBar, Quote, Fundamentals (18 sections), FundamentalMetric, PredictionMarket, PredictionMarketSnapshot

## Key Tables
securities, instruments, ohlcv_bars (hypertable), quotes, 18 fundamentals tables, fundamental_metrics (projection), ingestion_events, failed_tasks, outbox_events, prediction_markets, prediction_market_snapshots (hypertable)

## Prediction Markets (PRD-0019, Wave B-1 + B-2 + QA fixes)
- Consumer: `PredictionMarketConsumer` — group `market-data-prediction-markets`, topic `market.prediction.v1`
- Topic constant: `messaging.topics.MARKET_PREDICTION` (never hardcode the string)
- Upserts `prediction_markets` (one row per `market_id`) and inserts snapshots (`prediction_market_snapshots` hypertable, unique on `(market_id, snapshot_at)`)
- BP-034 event-id dedup via atomic `ingestion_events.create_if_not_exists` before any domain logic
- BP-122 Confluent Avro detection: checks for `0x00` magic byte; falls back to JSON
- **Wave B-2 API**: 3 read endpoints via `query_prediction_markets.py` use cases; all use `ReadOnlyUnitOfWork`
  - All 3 endpoints require `X-Internal-Token` header (D-02, R27 platform standard)
- `ListPredictionMarketsUseCase` avoids N+1 via `get_latest_prices_batch()` (DISTINCT ON query, 2 total queries)
- History endpoint: use case raises `ValueError` for `from_dt >= to_dt`; router maps to HTTP 400
- `volume_24h` not stored on market entity — returned as `None` in list/detail; per-snapshot in history
- `status=all` is translated to `None` (no filter) before repo call
- `PredictionMarketSnapshot.__post_init__` enforces `len(outcomes_prices) >= 2` — consumer pads to 2 entries if needed to avoid crash on malformed events
- `PredictionMarketSnapshot` is `frozen=True, slots=True` (memory optimization)
- Entry point: `market_data.infrastructure.messaging.consumers.prediction_market_consumer_main`
- **D-01 (migration 006)**: `prediction_markets` gained `last_snapshot_at TIMESTAMPTZ` column; `prediction_market_snapshots` PK changed from `(id)` to `(id, snapshot_at)` for TimescaleDB compliance
- **M-04**: Consumer `process_message` must NOT call `uow.commit()` — base class owns the single commit
- **storage_access_key/storage_secret_key** are now `SecretStr` — call `.get_secret_value()` where used

## Architecture Pattern (QA-013, Wave 3)
All read operations are wrapped in explicit use case classes under `application/use_cases/`:
- `query_instruments.py` — GetInstrumentByIdUseCase, GetInstrumentBySymbolUseCase, SearchInstrumentsUseCase
- `query_securities.py` — GetSecurityUseCase, ListSecuritiesUseCase
- `query_quotes.py` — GetQuoteUseCase, GetQuotesBatchUseCase
- `query_ohlcv.py` — GetOHLCVBarsUseCase, GetOHLCVBulkUseCase, GetAvailableTimeframesUseCase, GetOHLCVRangeUseCase
- `query_fundamentals.py` — GetFundamentalsSectionUseCase
- `query_fundamental_metrics.py` — GetFundamentalMetricsTimeseriesUseCase, ScreenInstrumentsUseCase, GetAvailableFundamentalMetricsUseCase
- `resample_ohlcv.py` — ResampledOHLCVUseCase (derive 5m/15m/30m/1h/4h bars from 1m triggers)

**API routers must never import from `infrastructure/`** (enforced by import guard IG-LAYER-002). Use cases receive an already-entered UoW from `get_uow` — do NOT use `async with self._uow:` inside use case methods.

### Read/Write UoW Split (R27)
Query use cases receive a `ReadOnlyUnitOfWork` (not `UnitOfWork`) via `get_read_uow` / `ReadUoWDep` in `api/dependencies.py`. `SqlAlchemyReadOnlyUnitOfWork` opens only the read session. Only mutation use cases use `UoWDep`.

### Port Interface Pattern (N-002)
`QuoteCachePort` ABC is defined in `application/ports/cache.py`. Routers depend on `QuoteCachePort` (not `QuoteCache`) injected via `get_quote_cache()` in `api/dependencies.py`. `QuoteCache` extends `QuoteCachePort` and is only referenced in the dependency factory. This keeps the API layer decoupled from infrastructure details.

### Metrics Auth (M-004)
`GET /metrics` requires `X-Internal-Token` header (validated via `InternalAuthDep` / `verify_internal_token` in `api/dependencies.py`). Uses `hmac.compare_digest` for timing-safe comparison. Returns 401 if token missing or invalid.

## Pitfalls
- Fundamentals path param is `instrument_id` (instrument UUID), not security ID — parameter was renamed from `security_id` in Wave E2-2
- Literal routes registered before path-param routes to avoid matching conflicts
- `fundamental_metrics` is derived projection; source of truth is 18 section tables
- Caching: quotes cached 5s via Valkey
- Quote.bid/ask/last/volume are nullable (`Decimal | None`, `int | None`) — NULL means "no data", 0 means "zero activity" (D-004)
- Consumer dedup uses atomic `create_if_not_exists()` — INSERT … ON CONFLICT DO NOTHING … RETURNING (BP-035). `is_duplicate()` always returns False; `mark_processed()` is a no-op.
- `event_type`/`schema_version` in domain events are ClassVar — not dataclass instance fields. Use `type(event).event_type` or `event.event_type`; `dataclasses.asdict()` excludes them so the outbox dispatcher adds them manually.
- Instrument search uses `_escape_like()` before building ILIKE pattern — escapes `%`, `_`, `\` to prevent wildcard injection (M-002)
- `get_uow` yields an already-entered UoW (via `async with ... as uow: yield uow`) — use cases must NOT call `async with self._uow:` again
- `is_partial=True` implies `is_derived=True` — domain entity `__post_init__` enforces this invariant; never set `is_partial=True` without `is_derived=True`
- Use `find_by_instrument_timeframe_datetime_range` (datetime args) for intraday period queries; `find_by_instrument_timeframe_range` (date args) is for daily/weekly ranges only

## Outbox / Event Emission Pattern (QA-016, Wave 5)

Consumers emit `InstrumentCreated` and `InstrumentUpdated` by writing **directly to `uow.outbox_events`** within the same DB transaction as domain writes. This is the canonical pattern for atomic event emission:

```python
from market_data.infrastructure.messaging.outbox.dispatcher import EVENT_TOPIC_MAP, event_to_outbox_payload

event = InstrumentCreated(instrument_id=..., symbol=..., exchange=...)
await uow.outbox_events.create(
    event_type=event.event_type,
    topic=EVENT_TOPIC_MAP[event.event_type],
    payload=event_to_outbox_payload(event),
)
```

Key points:
- **Do NOT use `uow.collect_event()`** for cross-service events — requires `outbox_notifier` to be wired (BP-058); not guaranteed in app.py
- **`EVENT_TOPIC_MAP`** routes each event type to its own dedicated topic — never to a shared/legacy topic (BP-059)
- **`event_to_outbox_payload()`** adds `entity_id = instrument_id` (M-017) and converts `tuple → list` for Avro arrays
- **Consumers must emit `InstrumentUpdated`** when `update_flags()` is called — not just `InstrumentCreated` (BP-061)
- **`InstrumentCreated.schema_version = 2`** since Wave 5: has optional `name`, `isin`, `instrument_type` fields
- **Canonical Avro schemas** in `infra/kafka/schemas/` (not service-local `infrastructure/messaging/schemas/`)

## Bug Patterns
- BP-034: mark_processed must be called before ANY early return — consumers must not return early before calling mark_processed (fixed in Wave E2-1)
- BP-035: Consumer dedup uses atomic INSERT … ON CONFLICT DO NOTHING … RETURNING; is_duplicate() always returns False; mark_processed() is a no-op
- BP-036: SELECT-FOR-UPDATE required when reading shared mutable state (budgets, watermarks) inside transactions
- BP-023: Use `uvx ruff format` (not venv ruff) for pre-commit formatting
- BP-058: `collect_event()` with no outbox_notifier — events silently lost
- BP-059: EVENT_TOPIC_MAP routing to wrong/shared topic — messages never reach consumers
- BP-060: Use `uow.outbox_events.create()` directly (atomic) instead of `collect_event()` (non-atomic)
- BP-061: Always emit `InstrumentUpdated` when `update_flags()` is called
- BP-062: Cross-service field name mismatch — `entity_id` vs `instrument_id` (use `event_to_outbox_payload()`)
- BP-063: Consumer using `json.loads()` for Avro-encoded messages (Confluent wire format)
- **PLAN-0017 screener**: `instruments` table column is `symbol` (not `ticker`). API response field is `ticker`. SQL must alias: `i.symbol AS ticker`. Never write `i.ticker` in raw SQL against this table.

## Intraday Resampling (PLAN-0040 Waves B-1..B-3)
- `IntradayResamplingWorker(BaseKafkaConsumer)` on group `market-data-intraday-resampling`, topic `market.dataset.fetched`
- Filters: only `dataset_type=ohlcv, timeframe=1m` events are processed; all others ACK'd immediately
- Calls `ResampledOHLCVUseCase` for each target timeframe in `_RESAMPLE_TARGETS = ["5m","15m","30m","1h","4h","1d"]`
- `_floor_to_period(bar_dt, period_seconds)`: `epoch_seconds = int(bar_dt.timestamp()); (epoch_seconds // period_seconds) * period_seconds`
- Partial-bar invariant: every resampled bar in an incomplete period has `is_partial=True, is_derived=True`
- `OHLCVBar.is_partial: bool = False` with `__post_init__` asserting `if self.is_partial and not self.is_derived: raise ValueError`
