# Market Data Service (S3) — Agent Context

**Port**: 8003 | **DB**: `market_data_db` (TimescaleDB) | **Status**: Mature

## Mission
Materialize OHLCV bars, quotes, and fundamentals from claim-check pointers. Serve query APIs for charts and financial data. Manage instrument lifecycle events.

## Domain Entities
Security, Instrument, OhlcvBar, Quote, Fundamentals (18 sections), FundamentalMetric

## Kafka Topics
| Direction | Topic | Notes |
|-----------|-------|-------|
| Produces | `market.instrument.created` | New instrument discovered |
| Produces | `market.instrument.updated` | Instrument metadata updated |
| Consumes | `market.dataset.fetched` | 3 consumer groups: ohlcv, quotes, fundamentals |

## Key Tables
securities, instruments, ohlcv_bars (hypertable), quotes, 18 fundamentals tables, fundamental_metrics (projection), ingestion_events, failed_tasks, outbox_events

## API Endpoints (30)
instruments, ohlcv (candles/charts), quotes, fundamentals (18 sections), securities

## Pitfalls
- Fundamentals path param is instrument UUID, not security ID
- Literal routes registered before path-param routes to avoid matching conflicts
- `fundamental_metrics` is derived projection; source of truth is 18 section tables
- Caching: quotes cached 5s via Valkey
