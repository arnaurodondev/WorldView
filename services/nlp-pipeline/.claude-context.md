# NLP Pipeline Service (S6) — Agent Context

**Port**: 8006 | **DB**: `nlp_db` (owned) + `intelligence_db` (read/write, DDL NOT owned) | **Status**: In-progress (C-1 + C-2 + C-3 done)

## Mission
Full intelligence enrichment: sectioning, GLiNER NER (**11 classes**), routing score, suppression, chunk/section embeddings (BGE 1024-dim), novelty gate, entity resolution cascade, deep LLM extraction (Qwen2.5-7B).

## Domain Entities
Section (with `speaker` field for transcripts), Chunk, EntityMention (with `resolution_stage`), MentionResolution (audit trail per cascade stage), DocumentEntityStats, ChunkEmbedding, SectionEmbedding, RoutingDecision (with `final_routing_tier`)

## Databases
- **nlp_db** (owned, Alembic enabled): sections (speaker), chunks, entity_mentions (resolution_stage), mention_resolutions, document_entity_stats, chunk_entity_mentions, chunk_embeddings, section_embeddings, routing_decisions (final_routing_tier), outbox_events, dead_letter_queue
- **intelligence_db** (DDL owned by intelligence-migrations): writes to canonical_entities, relation_evidence_raw, article_claims

## ML Models
- BGE embedding (1024-dim) via `libs/ml-clients`
- GLiNER NER (**11 classes**: organization, government_body, regulatory_body, financial_institution, person, financial_instrument, location, commodity, index, currency, **macroeconomic_indicator**)
- Qwen2.5-7B extraction via `libs/ml-clients`

## Suppression Gate Paths
- `SUPPRESS` (<0.20) → `ProcessingPath.HALT`
- `LIGHT` (0.20–0.45) → `ProcessingPath.SECTION_EMBEDDINGS_ONLY`
- `MEDIUM`/`DEEP` (≥0.45) → `ProcessingPath.FULL_PIPELINE`
- Uses `final_routing_tier` over `routing_tier` when set (novelty correction)

## API Endpoints (10)
signals, entities, entity detail, articles by entity, vector search, reprocess, topics

## Critical Rules
- **NEVER** run Alembic against `intelligence_db` from this service (`ALEMBIC_ENABLED=false`)
- Use `libs/ml-clients` for ALL ML calls (EmbeddingClient, NERClient, ExtractionClient)
- Backpressure via `asyncio.Semaphore` on Ollama queue depth
- Zero NER mentions NEVER suppress a document — explicit invariant, tested
- NMS IoU threshold is **strictly > 0.5** (not >=)

## Block 7-10 Key Behaviours (Wave C-3)
- **Block 7**: `chunk_section()` uses `_SENTENCE_END_RE` to split sentences; never splits mid-sentence. Section embeddings for ALL tiers; chunk embeddings only on FULL_PIPELINE.
- **Block 8**: MinHash key = `s5:minhash:article:<doc_id>` in Valkey; ONLY DEEP tier is downgraded (MEDIUM never touched by novelty gate). novelty_score returned for Block 5 feature vector.
- **Block 9**: `_stage4_ann` requires `clear_margin > 0.10` between top-1 and top-2 ANN results — ambiguous results discarded. UNRESOLVED mentions are NEVER deleted.
- **Block 10**: `ExtractionResult = dict[str, Any]`; evidence_date = `coalesce(published_at, extracted_at)` — never `now()`. Claims written to `nlp_db.outbox_events` via `ClaimsRepository` (never direct to `intelligence_db`).
- **Backpressure**: `BackpressureController(max_depth, resume_depth)` with `asyncio.Semaphore`. Context manager cleans up on exception. `is_paused` property used by Kafka consumer loop.

## Pitfalls
- HNSW indexes must include partial predicate `WHERE (expires_at IS NULL OR expires_at > now())` with embedding_status = 'ready' check
- Chunk and section embeddings use SEPARATE HNSW indexes
- Routing thresholds: deep >= 0.70, medium >= 0.45, light >= 0.20
- ORM: `MentionResolutionModel.metadata` column Python attr renamed to `resolution_metadata` (column name stays `metadata`) to avoid collision with SQLAlchemy `DeclarativeBase.metadata`
- Watchlist Valkey ops are best-effort — always return 0.0 on unavailability, never raise
- `BackpressureController.resume_depth` MUST be < `max_depth` — constructor raises `ValueError` otherwise
- `_split_sentences` uses `re.compile(r"(?<=[.!?])\s+")` — text with no punctuation becomes 1 sentence
