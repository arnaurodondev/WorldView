# NLP Pipeline Service (S6) ‚Äî Agent Context

**Port**: 8006 | **DB**: `nlp_db` (owned) + `intelligence_db` (read/write, DDL NOT owned) | **Status**: Feature-complete (all blocks 3‚Äì10 implemented)

## Mission
Full intelligence enrichment: sectioning (Block 3), GLiNER NER ‚Äî **11 classes** (Block 4), routing score 8-signal weighted (Block 5), suppression gate (Block 6), chunk/section embeddings BGE 1024-dim (Block 7), novelty gate MinHash+embedding (Block 8), entity resolution cascade 4-stage (Block 9), deep LLM extraction (Block 10).

## Domain Entities
- `Section` ‚Äî with `speaker` field for transcript source types
- `Chunk` ‚Äî with `chunk_text_key: str | None` (MinIO object key from Block 7); `entity_mentions: list[dict]` JSONB (GIN-indexed) populated after Block 9 resolution; each dict: `{entity_id, entity_type, char_start, char_end, gliner_score, raw_text}`
- `EntityMention` ‚Äî with `resolution_stage` (1‚Äì4), `ner_model_id: str | None`
- `MentionResolution` ‚Äî per-stage audit trail for resolution cascade
- `DocumentEntityStats` ‚Äî aggregated NER stats per document
- `ChunkEmbedding`, `SectionEmbedding` ‚Äî 1024-dim vectors with `expires_at` for HNSW partial index
- `RoutingDecision` ‚Äî with `final_routing_tier` (set after Block 8 novelty correction), `composite_score`, `feature_scores_json`, and (PLAN-0111 C-6 SHADOW) `learned_tier`/`learned_p_yield`/`learned_router_mode` ‚Äî the EmbeddingGemma learned-router proposal; observational only, never controls processing
- `DocumentSourceMetadata` ‚Äî citation cache for S8 RAG; includes `llm_relevance_score`, `llm_scored_at`, `sentiment`, `impact_score`
- `ArticleImpactWindow` ‚Äî multi-window price-impact labels (replaces `article_price_impacts`); one row per `(article_id, entity_id, window_type)`

## Critical Rules
- **NEVER** run Alembic against `intelligence_db` ‚Äî `ALEMBIC_ENABLED=false` for that DB
- Use `libs/ml-clients` for ALL ML calls (EmbeddingClient, NERClient, ExtractionClient)
- Backpressure via `BackpressureController(max_depth, resume_depth)` ‚Äî `resume_depth MUST be < max_depth`
- Zero NER mentions NEVER suppress a document ‚Äî explicit invariant, tested
- NMS IoU threshold is **strictly > 0.5** (not >=)
- `entity.provisional.queued.v1` is published via outbox, NOT `entity.canonical.created.v1` (S7 emits the canonical created event)
- Claims flow via `nlp.article.enriched.v1.raw_claims_json` ‚Äî the legacy `claim.extracted` topic was removed (PLAN-0057 D-1)
- `article_impact_windows` replaces `article_price_impacts` ‚Äî always use the new table

## Key Pitfalls
- **HNSW indexes must use partial predicate**: `WHERE (expires_at IS NULL OR expires_at > now())`; use `op.execute()` in Alembic (not native HNSW support)
- **Chunk and section embeddings use SEPARATE HNSW indexes** ‚Äî mixing granularities pollutes ANN recall
- **`routing_decisions.composite_score` is the routing score** ‚Äî do NOT duplicate into `document_source_metadata`; access via JOIN on `doc_id`
- **Learned router modes off|shadow|live (PLAN-0111 C-6/C-7/C-8)** ‚Äî `NLP_PIPELINE_LEARNED_ROUTER_MODE`. `LearnedRouter` (`application/blocks/learned_routing.py`) proposes a tier per article (logged `learned_router_shadow` + counted + persisted to `learned_*` columns). In `shadow` it NEVER changes routing. In `live` (C-8) the post-cascade learned tier CONTROLS processing: `_apply_live_learned_tier` writes it onto `final_routing_tier` (which `apply_suppression_gate` + downstream gates read); the STATIC tier stays in `routing_tier` for comparison. **Three live invariants**: (1) title-less docs (blind classifier ‚Äî reads title+lede) fall back to the static tier (`learned_router_titleless_fallback`); (2) static SUPPRESS is never resurrected; (3) regulatory/`tenant_upload` sources still forced ‚â•MEDIUM. **C-7 cascade** (`NLP_PIPELINE_LEARNED_ROUTER_CASCADE`): in-band `P(yield)‚àà[0.48,0.68]` escalates to the REUSED Llama-8B relevance scorer (`relevance_scoring_api_model_id` + `ARTICLE_RELEVANCE_SCORER` prompt ‚Äî `relevance_cascade.py`); relevance‚â•0.5‚ÜíMEDIUM else LIGHT; out-of-band skips the LLM. Model artifact + meta in `application/blocks/models/`; LightGBM needs `libgomp1` (Dockerfile runtime).
- **`ArticleRelevanceScoringWorker` must follow R24 3-phase**: read batch ‚Üí close session ‚Üí Ollama calls ‚Üí open new session ‚Üí batch UPDATE; never hold DB session open during Ollama calls
- **Worker routing tier filter uses `COALESCE(rd.final_routing_tier, rd.routing_tier)`** ‚Äî novelty-downgraded articles have `final_routing_tier='LIGHT'` but `routing_tier='DEEP'`
- **`day_t2/day_t5` are cumulative from `close_t0`** ‚Äî fetch day_t0 bar first; use its close as baseline
- **`DisplayRelevanceScore.value` uses `is not None` not `or 0.0`** ‚Äî `mi=0.0` (zero price movement) and `mi=None` (not yet labelled) have different semantics
- **`NLP_PIPELINE_SERVICE_ACCOUNT_TOKEN` required in production** (BP-303) ‚Äî without it, PriceImpactLabellingWorker falls back to `dev-login` which is blocked in production
- **Ollama JSON mode**: pass `"format": "json"` in API body ‚Äî prompt-only instruction is insufficient
- **`UNNEST pg_trgm batch`**: use `bindparam('terms', type_=sa.ARRAY(sa.Text()))` not `::type` cast syntax (asyncpg rejects it, BP-076)
- **`FailureInfo[None]` has NO `value`/`key`/`headers` fields** ‚Äî only `event_id`, `topic`, `partition`, `offset`, `attempt`, `last_error`, `record`
- **CanonicalTickersCache atomic swap uses Lua script** (`_ATOMIC_TICKER_SWAP`) not MULTI/EXEC ‚Äî eliminates partial-clear race on network drop between DEL and SADD (BP-422)

## Endpoint recovery (2026-06-14 entity-ref-matching mitigation ‚Äî relation-drop fix)
- `_enqueue_enriched` now runs `recover_missed_endpoints` (`consumers/blocks/endpoint_recovery.py`) BEFORE `_build_raw_*` for refs that miss the doc-local `entity_id_by_ref`. Two layers, applied uniformly to relations/events/claims:
  - **M1** ‚Äî batched canonical-store fall-back via `EntityAliasRepository.batch_exact_match` + `batch_ticker_isin_match` behind the 0.75 floor + 0.15 delta gate ‚Üí binds to a REAL canonical (`entity_provisional=False`). ONE query per stage for N missed refs.
  - **M2** ‚Äî `ensure_provisional_for_ref` (`application/blocks/entity_resolution.py`) mints a `provisional_entity_queue` row for the still-unresolved (non-junk) endpoint ‚Üí row persists with `entity_provisional=True`; `UnresolvedResolutionWorker` canonicalizes later ‚Üí KG promotion. Junk guard (`_is_junk_ref`) blocks common-noun/empty/numeric refs.
- Final drop is now reserved for refs that are empty/junk/hallucinated after BOTH M1 and M2. Observability: `s6_extraction_endpoint_recovery_total{outcome=m1_recovered|m2_minted|dropped_junk}` ‚Äî all three outcome series are **seeded to 0 at module import** in `metrics/prometheus.py` so they appear at `/metrics` (port 9100) from process boot; a labelled prometheus counter exports NO series for an un-incremented label, which is why the metric looked "missing" before seeding.
- Distinct from `synthesize_provisional_refs` (which only promotes refs that are already NER mentions); M2 rescues the dominant miss ‚Äî the non-mention counterparty endpoint GLiNER never tagged.
