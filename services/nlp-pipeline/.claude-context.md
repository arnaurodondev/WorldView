# NLP Pipeline Service (S6) — Agent Context

**Port**: 8006 | **DB**: `nlp_db` (owned) + `intelligence_db` (read/write, DDL NOT owned) | **Status**: Compliant

## Mission
Full intelligence enrichment: sectioning, GLiNER NER (10 classes), routing score, suppression, chunk/section embeddings (BGE 1024-dim), novelty gate, entity resolution cascade, deep LLM extraction (Qwen2.5-7B).

## Domain Entities
Section, Chunk, EntityMention, ChunkEmbedding, SectionEmbedding, RoutingDecision

## Kafka Topics
| Direction | Topic | Notes |
|-----------|-------|-------|
| Produces | `nlp.article.enriched.v1` | Fully enriched article |
| Produces | `nlp.signal.detected.v1` | Detected financial signals |
| Consumes | `content.article.stored.v1` | Cleaned articles from S5 |

## Databases
- **nlp_db** (owned, Alembic enabled): sections, chunks, entity_mentions, chunk_entity_mentions, chunk_embeddings, section_embeddings, routing_decisions, outbox_events, dead_letter_queue
- **intelligence_db** (DDL owned by intelligence-migrations): writes to canonical_entities, relation_evidence_raw, article_claims

## ML Models
- BGE embedding (1024-dim) via `libs/ml-clients`
- GLiNER NER (10 classes: organization, government_body, regulatory_body, financial_institution, person, financial_instrument, location, commodity, index, currency)
- Qwen2.5-7B extraction via `libs/ml-clients`

## API Endpoints (10)
signals, entities, entity detail, articles by entity, vector search, reprocess, topics

## Critical Rules
- **NEVER** run Alembic against `intelligence_db` from this service (`ALEMBIC_ENABLED=false`)
- Use `libs/ml-clients` for ALL ML calls (EmbeddingClient, NERClient, ExtractionClient)
- Backpressure via `asyncio.Semaphore` on Ollama queue depth

## Pitfalls
- HNSW indexes must include partial predicate `WHERE (expires_at IS NULL OR expires_at > now())`
- Chunk and section embeddings use SEPARATE HNSW indexes
- Routing thresholds: deep >= 0.70, medium >= 0.45, light >= 0.20
