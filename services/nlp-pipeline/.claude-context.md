## Block 7-10 Key Behaviours (Wave C-3)
- **Block 7**: `chunk_section()` uses `_SENTENCE_END_RE` to split sentences; never splits mid-sentence. Section embeddings for ALL tiers; chunk embeddings only on FULL_PIPELINE. **Chunk text upload to MinIO for ALL tiers** (Option B): after all chunks are created, `_upload_chunk_texts()` fires `asyncio.gather` to upload each chunk text as `nlp-pipeline/chunk-text/{doc_id}/{chunk_id}/body/v1.txt` in `NLP_PIPELINE_CHUNK_BUCKET`. Upload failure is logged as a warning and sets `text_key=None` on the chunk â€” never raises. `Chunk` is a frozen dataclass so updates use `dataclasses.replace(chunk, text_key=key)`. Port: `ChunkTextStorePort`; adapter: `MinIOChunkTextStore`.
- **Block 8**: MinHash key = `s5:minhash:article:<doc_id>` in Valkey; ONLY DEEP tier is downgraded (MEDIUM never touched by novelty gate). novelty_score returned for Block 5 feature vector.
- **Block 9 (batch-optimised)**: 3 DB round-trips regardless of mention count (1 per batch stage):
  - Stage 1: `alias_repo.batch_exact_match(all_texts)` â†’ `dict[str, UUID]` (IN clause)
  - Stage 2: `alias_repo.batch_ticker_isin_match(tickers, isins)` â†’ `dict[str, UUID]` (1-2 queries)
  - Stage 3: `alias_repo.batch_fuzzy_trigram(stage3_texts, threshold, top_k)` â†’ `dict[str, list[tuple[UUID,float]]]` (UNNEST lateral join)
  - Stage 4 (ANN HNSW): per-mention only for unresolved after stages 1-3; requires `clear_margin > 0.10`
  - UNRESOLVED mentions are NEVER deleted; stages 1-2 emit miss audit entries for complete trail
- **Block 10**: `ExtractionResult = dict[str, Any]`; evidence_date for downstream consumers = `coalesce(published_at, extracted_at)` carried in the enriched envelope â€” never `now()`. Claims flow downstream as the `raw_claims` array on `nlp.article.enriched.v1`; the legacy `claim.extracted` producer (PLAN-0057 D-1, F-CRIT-08) was removed because the topic had zero subscribers.
- **Backpressure**: `BackpressureController(max_depth, resume_depth)` with `asyncio.Semaphore`. Context manager cleans up on exception. `is_paused` property used by Kafka consumer loop.

## Pitfalls
- HNSW indexes must include partial predicate `WHERE (expires_at IS NULL OR expires_at > now())` with embedding_status = 'ready' check
- Chunk and section embeddings use SEPARATE HNSW indexes
- Routing thresholds: deep >= 0.70, medium >= 0.45, light >= 0.20
- ORM: `MentionResolutionModel.metadata` column Python attr renamed to `resolution_metadata` (column name stays `metadata`) to avoid collision with SQLAlchemy `DeclarativeBase.metadata`
- Watchlist Valkey ops are best-effort â€” always return 0.0 on unavailability, never raise
- `BackpressureController.resume_depth` MUST be < `max_depth` â€” constructor raises `ValueError` otherwise
- `_split_sentences` uses `re.compile(r"(?<=[.!?])\s+")` â€” text with no punctuation becomes 1 sentence
- **`routing_decisions` stores routing_score, NOT `document_source_metadata`**: `routing_decisions.composite_score` is the routing score; access via JOIN on `doc_id`. Add `idx_routing_decisions_doc_id` index (PRD-0026 migration) for efficient JOIN. Do NOT add routing_score to document_source_metadata.
- **`article_impact_windows` replaces `article_price_impacts`** (PRD-0026): always write to and read from `article_impact_windows`; use `window_type = 'day_t0'` for backward-compatible single-window access (e.g., for `nlp.signal.detected.v1` emission).
- **NER cache stores raw spans, not EntityMention** *(pending PRD-0023)*: Cache key = `nlp:ner_cache:v1:{sha256(all_section_texts)}`. Cached value = JSON list of `{text, label, score, start, end}` (pre-NMS raw GLiNER output). On cache hit, reconstruct `EntityMention` with fresh UUIDs and current `doc_id`/`section_id`. NMS still runs on reconstructed mentions.
- **NER cache is best-effort** *(pending PRD-0023)*: All Valkey calls wrapped in `try/except Exception`; on error, log warning and fall through to GLiNER. Never raises. Feature-flagged via `NLP_PIPELINE_NER_CACHE_ENABLED`.
- **NER cache SHA256 covers concatenated text** *(pending PRD-0023)*: Hash = `hashlib.sha256("\n".join(s.text for s in sections).encode()).hexdigest()`. The join separator is `"\n"` â€” consistent across calls for the same section list.
- **`DisplayRelevanceScore.value` uses `is not None` not `or 0.0`** *(PRD-0026)*: `mi = self.market_impact` (NOT `mi = self.market_impact or 0.0`). `mi=0.0` (zero price movement) and `mi=None` (not yet labelled) have different semantics. Use `if mi is not None and mi > 0`.
- **`ArticleRelevanceScoringWorker` must follow R24 (3-phase)**: Read batch â†’ close session â†’ Ollama calls â†’ open new session â†’ batch UPDATE. Never hold DB session open during Ollama HTTP calls.
- **Worker routing tier filter uses `COALESCE(rd.final_routing_tier, rd.routing_tier)`**: Articles downgraded by novelty gate have `final_routing_tier='LIGHT'` but `routing_tier='DEEP'`. Using only `routing_tier` incorrectly scores novelty-suppressed articles.
- **day_t2/day_t5 are cumulative from `close_t0`**: `price_start = close_t0` from the day_t0 OHLCV bar (NOT the open of day_t2/t5). Fetch day_t0 bar first; use its close as baseline. If day_t0 returns 404, skip all cumulative windows.
- **Migration must filter `WHERE price_open > 0`**: `article_price_impacts` has zero-sentinel rows from `ArticlePriceImpact.zero()`. These represent missing data â€” do NOT migrate them.
- **`article_impact_windows` idempotency key needs `UNIQUE INDEX`**: `ON CONFLICT (article_id, entity_id, window_type) DO NOTHING` requires a UNIQUE index, not a plain index. Use `CREATE UNIQUE INDEX`.
- **Ollama JSON mode**: Pass `"format": "json"` in the Ollama API request body for constrained JSON output. Prompt-only instruction is insufficient.
- **Score discontinuity is intentional**: Article may rank lower after day_t0 market data confirms no price movement vs. its pre-market LLM-predicted score. See PRD-0026 Â§7 AD-5.

## Canonical Tickers Cache (PLAN-0084 C-1)

- **Key**: `nlp:v1:canonical_tickers` (Valkey SET, shared across replicas)
- **Env var**: `NLP_PIPELINE_CANONICAL_TICKERS_REFRESH_INTERVAL_S` â€” refresh interval in seconds (default 600, range 60-3600)
- **Atomic swap**: `refresh()` uses a Lua script (`_ATOMIC_TICKER_SWAP`: DEL + SADD as a single server-side atomic op) â€” eliminates partial-failure cache-wipe on network drop between DEL and SADD (BP-422 CLOSED)
- **Backoff**: Valkey write failures in `refresh()` re-raise so `_refresh_loop()` applies exponential backoff (`min(2^n * 60, 300)s`) and increments `_REFRESH_FAILURE_COUNTER` (F-DS009 CLOSED)
- **Background loop**: `startup()` â†’ `refresh()` once, then `asyncio.create_task(_refresh_loop())` for periodic refresh
- **Close**: `close()` cancels and awaits the task; safe before `startup()`
- **Staleness**: at most `canonical_tickers_refresh_interval_s` seconds stale (default: 600s = 10 min)
- **Source**: `intelligence_db.canonical_entities WHERE entity_type='financial_instrument' AND ticker IS NOT NULL`

## Chunk Text Retrieval (Option B â€” MinIO, Wave A-1 extension)

`EnhancedChunkSearchUseCase._fetch_chunk_texts()` enriches search results with full chunk text:
1. Builds `key_map: dict[UUID, str]` â€” only for results with `chunk_text_key` AND `granularity == "chunk"` (sections always fall back to `heading_path`).
2. Early-returns `{}` if `key_map` is empty (no I/O).
3. Checks Valkey cache at `nlp:v1:chunk_text:{chunk_id}` (1h TTL).
4. Calls `chunk_text_store.get_batch(uncached)` for cache misses only (skips call if `uncached` is empty).
5. Writes fetched texts to Valkey cache.
6. On `get_batch` failure: logs warning, falls back to `heading_path or ""` (never raises).
7. When `chunk_text_store is None`: text stays as `heading_path` â€” backwards compatible.

Config: `NLP_PIPELINE_CHUNK_BUCKET` (default `"worldview"`), `NLP_PIPELINE_STORAGE_ACCESS_KEY` (empty = MinIO disabled, store stays None).

- **D-004 FIXED (PLAN-0031 B-3)**: Both `nlp_session` and `intel_session` are opened at the top of `_run_pipeline()`. NLP commits FIRST; if it fails, intel rolls back via context manager `__aexit__`. Asyncpg rejects `::type` cast syntax in `text()` params â€” use `bindparam('terms', type_=sa.ARRAY(sa.Text()))` if `text()` casts fail (BP-076).
