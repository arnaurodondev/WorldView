# NLP Pipeline Service (S6) — Agent Context

**Port**: 8006 | **DB**: `nlp_db` (owned) + `intelligence_db` (read/write, DDL NOT owned) | **Status**: Feature-complete (all blocks 3–10 implemented)

## Mission
Full intelligence enrichment: sectioning (Block 3), GLiNER NER — **11 classes** (Block 4), routing score 8-signal weighted (Block 5), suppression gate (Block 6), chunk/section embeddings BGE 1024-dim (Block 7), novelty gate MinHash+embedding (Block 8), entity resolution cascade 4-stage (Block 9), deep LLM extraction (Block 10).

## Domain Entities
- `Section` — with `speaker` field for transcript source types
- `Chunk` — with `chunk_text_key: str | None` (MinIO object key from Block 7); `entity_mentions: list[dict]` JSONB (GIN-indexed) populated after Block 9 resolution; each dict: `{entity_id, entity_type, char_start, char_end, gliner_score, raw_text}`
- `EntityMention` — with `resolution_stage` (1–4), `ner_model_id: str | None`
- `MentionResolution` — per-stage audit trail for resolution cascade
- `DocumentEntityStats` — aggregated NER stats per document
- `ChunkEmbedding`, `SectionEmbedding` — 1024-dim vectors with `expires_at` for HNSW partial index
- `RoutingDecision` — with `final_routing_tier` (set after Block 8 novelty correction), `composite_score`, `feature_scores_json`
- `DocumentSourceMetadata` — citation cache for S8 RAG; includes `llm_relevance_score`, `llm_scored_at`, `sentiment`, `impact_score`
- `ArticleImpactWindow` — multi-window price-impact labels (replaces `article_price_impacts`); one row per `(article_id, entity_id, window_type)`

## Databases
- **nlp_db** (owned, Alembic enabled): sections, chunks (`chunk_text_key`, `entity_mentions` JSONB GIN index), entity_mentions (`resolution_stage`, `ner_model_id`), mention_resolutions, document_entity_stats, chunk_entity_mentions, chunk_embeddings (HNSW partial index), section_embeddings (HNSW partial index), routing_decisions (`final_routing_tier`, `composite_score`, `feature_scores_json`), document_source_metadata (`llm_relevance_score`, `llm_scored_at`, `sentiment`, `impact_score`), article_impact_windows (`UNIQUE (article_id, entity_id, window_type)`), outbox_events, dead_letter_queue, llm_usage_log

## ML Adapters
- **GLiNER** (11 classes): `GLiNERHTTPAdapter(base_url, semaphore)` when `NLP_PIPELINE_GLINER_BASE_URL` is set; `GLiNERLocalAdapter(model_path, semaphore)` fallback
- **Embedding** (1024-dim): `DeepInfraEmbeddingAdapter` when `embedding_provider=deepinfra` + `EMBEDDING_API_KEY`; `JinaEmbeddingAdapter` when `embedding_provider=jina` + `JINA_API_KEY`; `OllamaEmbeddingAdapter` default
- **Deep extraction**: `DeepSeekExtractionAdapter(api_key, model_id, base_url, semaphore)` when `EXTRACTION_API_KEY` set; `OllamaExtractionAdapter` fallback
- **Relevance scoring**: DeepInfra (`RELEVANCE_SCORING_API_KEY`) or Ollama (`qwen3:0.6b`)
- **Unresolved resolution**: DeepInfra (`UNRESOLVED_RESOLUTION_API_KEY`) or Ollama (`qwen3:0.6b`)

## Routing Tier Thresholds
- DEEP ≥ 0.70
- MEDIUM ≥ 0.35 (lowered from 0.45 — watchlist fires post-resolution)
- LIGHT ≥ 0.20
- SUPPRESS < 0.20 → `ProcessingPath.HALT`

## Critical Rules
- **NEVER** run Alembic against `intelligence_db` — `ALEMBIC_ENABLED=false` for that DB
- Use `libs/ml-clients` for ALL ML calls (EmbeddingClient, NERClient, ExtractionClient)
- Backpressure via `BackpressureController(max_depth, resume_depth)` — `resume_depth MUST be < max_depth`
- Zero NER mentions NEVER suppress a document — explicit invariant, tested
- NMS IoU threshold is **strictly > 0.5** (not >=)
- `entity.provisional.queued.v1` is published via outbox, NOT `entity.canonical.created.v1` (S7 emits the canonical created event)
- Claims flow via `nlp.article.enriched.v1.raw_claims_json` — the legacy `claim.extracted` topic was removed (PLAN-0057 D-1)
- `article_impact_windows` replaces `article_price_impacts` — always use the new table

## Key Pitfalls
- **Design decision: enriched vs. temporal event filtering (DP-F001)**: Enriched events drop mentions that resolve to neither a canonical entity nor a provisional queue id — they would have no edge to anchor (see `infrastructure/messaging/consumers/blocks/enriched_event.py` `_build_raw_*`). Temporal events keep all events regardless of resolvable participants — macro events (CPI, FOMC, sanctions) legitimately have none, and S7's `TemporalEventConsumer` accepts `exposed_entities=[]`. Do not "fix" this asymmetry; it is intentional and was flagged DP-F001 in `docs/audits/2026-05-24-investigation-qa-open-items.md`.
- **HNSW indexes must use partial predicate**: `WHERE (expires_at IS NULL OR expires_at > now())`; use `op.execute()` in Alembic (not native HNSW support)
- **Chunk and section embeddings use SEPARATE HNSW indexes** — mixing granularities pollutes ANN recall
- **`routing_decisions.composite_score` is the routing score** — do NOT duplicate into `document_source_metadata`; access via JOIN on `doc_id`
- **Routing signal v2 (PLAN-0093 C-1, 2026-05-23)**: `compute_routing_score()` accepts ONLY 5 live signals — `entity_density` (0.35), `source_reliability` (0.30), `recency` (0.15), `document_type` (0.10), `extraction_yield` (0.10). The 3 dead signals (`watchlist`/`novelty`/`price_impact`) were dropped because they could never fire in single-pass routing. Tier thresholds recalibrated: DEEP=0.75, MEDIUM=0.45, LIGHT=0.20. Re-adding the dropped signals requires two-pass routing first.
- **entity_mentions persistence floor (PLAN-0093 C-2, 2026-05-23, F-NPL-005)**: `Settings.min_persist_floor` (default 0.6, env `NLP_PIPELINE_MIN_PERSIST_FLOOR`) gates writes to the `entity_mentions` table — sub-floor mentions are dropped at `persist_artifacts()` write time AND filtered out by `EntityMentionRepository.get_unresolved_batch{,_with_context}()` SQL (`WHERE confidence >= :min_confidence`) for defense-in-depth. `chunks.entity_mentions` JSONB still uses `gliner_mention_floor` (same default 0.6) — both should stay in sync.
- **Unknown-ticker skip-set (PLAN-0093 C-3, 2026-05-23, F-NPL-FUNDAMENTALS-001)**: `MarketDataClient._resolve_instrument_id` accepts an optional `valkey_client`; after 3 consecutive 404s for a ticker, the client writes a 7-day marker at `nlp:price_impact:unknown_tickers:<TICKER>` so subsequent worker cycles skip the round-trip. Per-ticker fail counter at `nlp:price_impact:unknown_tickers:fails:<TICKER>` with 1-day rolling TTL. All Valkey ops are best-effort — a Valkey outage falls through to the network path (degraded but functional). Wire the Valkey client when constructing `MarketDataClient` in production; tests + dev workflows pass `valkey_client=None`.
- **impact_score writer (PLAN-0093 C-3, 2026-05-23, F-NPL-008)**: `ArticleRelevanceScoringWorker` now writes `document_source_metadata.impact_score = MAX(impact_score)` from `article_impact_windows` in the same phase-3 session (single UPDATE…FROM batched per cycle). Stays NULL when no `article_impact_windows` row exists (PRD-0026 §6.5 — "not yet labelled" vs "zero impact" must remain distinguishable in the weighted-display formula).
- **`ArticleRelevanceScoringWorker` must follow R24 3-phase**: read batch → close session → Ollama calls → open new session → batch UPDATE; never hold DB session open during Ollama calls
- **Worker routing tier filter uses `COALESCE(rd.final_routing_tier, rd.routing_tier)`** — novelty-downgraded articles have `final_routing_tier='LIGHT'` but `routing_tier='DEEP'`
- **`day_t2/day_t5` are cumulative from `close_t0`** — fetch day_t0 bar first; use its close as baseline
- **`DisplayRelevanceScore.value` uses `is not None` not `or 0.0`** — `mi=0.0` (zero price movement) and `mi=None` (not yet labelled) have different semantics
- **`NLP_PIPELINE_SERVICE_ACCOUNT_TOKEN` required in production** (BP-303) — without it, PriceImpactLabellingWorker falls back to `dev-login` which is blocked in production
- **Ollama JSON mode**: pass `"format": "json"` in API body — prompt-only instruction is insufficient
- **`UNNEST pg_trgm batch`**: use `bindparam('terms', type_=sa.ARRAY(sa.Text()))` not `::type` cast syntax (asyncpg rejects it, BP-076)
- **`FailureInfo[None]` has NO `value`/`key`/`headers` fields** — only `event_id`, `topic`, `partition`, `offset`, `attempt`, `last_error`, `record`
- **CanonicalTickersCache atomic swap uses Lua script** (`_ATOMIC_TICKER_SWAP`) not MULTI/EXEC — eliminates partial-clear race on network drop between DEL and SADD (BP-422)

## Canonical Tickers Cache (PLAN-0084)
- **Key**: `nlp:v1:canonical_tickers` (Valkey SET)
- **Source**: `intelligence_db.canonical_entities WHERE entity_type='financial_instrument' AND ticker IS NOT NULL`
- **Refresh interval**: `NLP_PIPELINE_CANONICAL_TICKERS_REFRESH_INTERVAL_S` (default 600s, range 60–3600)
- **Atomic swap**: Lua script (DEL + SADD server-side atomic)
- **Backoff on Valkey failure**: `min(2^n × 60, 300)s` + `_REFRESH_FAILURE_COUNTER` Prometheus counter
