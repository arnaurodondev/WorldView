## Mission

## Block 7-10 Key Behaviours (Wave C-3)
- **Block 7**: `chunk_section()` uses `_SENTENCE_END_RE` to split sentences; never splits mid-sentence. Section embeddings for ALL tiers; chunk embeddings only on FULL_PIPELINE. **Chunk text upload to MinIO for ALL tiers** (Option B): after all chunks are created, `_upload_chunk_texts()` fires `asyncio.gather` to upload each chunk text as `nlp-pipeline/chunk-text/{doc_id}/{chunk_id}/body/v1.txt` in `NLP_PIPELINE_CHUNK_BUCKET`. Upload failure is logged as a warning and sets `text_key=None` on the chunk ‚Äî never raises. `Chunk` is a frozen dataclass so updates use `dataclasses.replace(chunk, text_key=key)`. Port: `ChunkTextStorePort`; adapter: `MinIOChunkTextStore`.
- **Block 8**: MinHash key = `s5:minhash:article:<doc_id>` in Valkey; ONLY DEEP tier is downgraded (MEDIUM never touched by novelty gate). novelty_score returned for Block 5 feature vector.
- **Block 9 (batch-optimised)**: 3 DB round-trips regardless of mention count (1 per batch stage):
  - Stage 1: `alias_repo.batch_exact_match(all_texts)` ‚Üí `dict[str, UUID]` (IN clause)
  - Stage 2: `alias_repo.batch_ticker_isin_match(tickers, isins)` ‚Üí `dict[str, UUID]` (1-2 queries)
  - Stage 3: `alias_repo.batch_fuzzy_trigram(stage3_texts, threshold, top_k)` ‚Üí `dict[str, list[tuple[UUID,float]]]` (UNNEST lateral join)
  - Stage 4 (ANN HNSW): per-mention only for unresolved after stages 1-3; requires `clear_margin > 0.10`
  - UNRESOLVED mentions are NEVER deleted; stages 1-2 emit miss audit entries for complete trail
- **Block 10**: `ExtractionResult = dict[str, Any]`; evidence_date = `coalesce(published_at, extracted_at)` ‚Äî never `now()`. Claims written to `nlp_db.outbox_events` via `ClaimsRepository` (never direct to `intelligence_db`).
- **Backpressure**: `BackpressureController(max_depth, resume_depth)` with `asyncio.Semaphore`. Context manager cleans up on exception. `is_paused` property used by Kafka consumer loop.

## Pitfalls
- HNSW indexes must include partial predicate `WHERE (expires_at IS NULL OR expires_at > now())` with embedding_status = 'ready' check
- Chunk and section embeddings use SEPARATE HNSW indexes
- Routing thresholds: deep >= 0.70, medium >= 0.45, light >= 0.20
- ORM: `MentionResolutionModel.metadata` column Python attr renamed to `resolution_metadata` (column name stays `metadata`) to avoid collision with SQLAlchemy `DeclarativeBase.metadata`
- Watchlist Valkey ops are best-effort ‚Äî always return 0.0 on unavailability, never raise
- `BackpressureController.resume_depth` MUST be < `max_depth` ‚Äî constructor raises `ValueError` otherwise
- `_split_sentences` uses `re.compile(r"(?<=[.!?])\s+")` ‚Äî text with no punctuation becomes 1 sentence
- **`routing_decisions` stores routing_score, NOT `document_source_metadata`**: `routing_decisions.composite_score` is the routing score; access via JOIN on `doc_id`. Add `idx_routing_decisions_doc_id` index (PRD-0026 migration) for efficient JOIN. Do NOT add routing_score to document_source_metadata.
- **`article_impact_windows` replaces `article_price_impacts`** (PRD-0026): always write to and read from `article_impact_windows`; use `window_type = 'day_t0'` for backward-compatible single-window access (e.g., for `nlp.signal.detected.v1` emission).
- **NER cache stores raw spans, not EntityMention** *(pending PRD-0023)*: Cache key = `nlp:ner_cache:v1:{sha256(all_section_texts)}`. Cached value = JSON list of `{text, label, score, start, end}` (pre-NMS raw GLiNER output). On cache hit, reconstruct `EntityMention` with fresh UUIDs and current `doc_id`/`section_id`. NMS still runs on reconstructed mentions.
- **NER cache is best-effort** *(pending PRD-0023)*: All Valkey calls wrapped in `try/except Exception`; on error, log warning and fall through to GLiNER. Never raises. Feature-flagged via `NLP_PIPELINE_NER_CACHE_ENABLED`.
- **NER cache SHA256 covers concatenated text** *(pending PRD-0023)*: Hash = `hashlib.sha256("\n".join(s.text for s in sections).encode()).hexdigest()`. The join separator is `"\n"` ‚Äî consistent across calls for the same section list.
- **`DisplayRelevanceScore.value` uses `is not None` not `or 0.0`** *(PRD-0026)*: `mi = self.market_impact` (NOT `mi = self.market_impact or 0.0`). `mi=0.0` (zero price movement) and `mi=None` (not yet labelled) have different semantics. Use `if mi is not None and mi > 0`.
- **`ArticleRelevanceScoringWorker` must follow R24 (3-phase)**: Read batch ‚Üí close session ‚Üí Ollama calls ‚Üí open new session ‚Üí batch UPDATE. Never hold DB session open during Ollama HTTP calls.
- **Worker routing tier filter uses `COALESCE(rd.final_routing_tier, rd.routing_tier)`**: Articles downgraded by novelty gate have `final_routing_tier='LIGHT'` but `routing_tier='DEEP'`. Using only `routing_tier` incorrectly scores novelty-suppressed articles.
- **day_t2/day_t5 are cumulative from `close_t0`**: `price_start = close_t0` from the day_t0 OHLCV bar (NOT the open of day_t2/t5). Fetch day_t0 bar first; use its close as baseline. If day_t0 returns 404, skip all cumulative windows.
- **Migration must filter `WHERE price_open > 0`**: `article_price_impacts` has zero-sentinel rows from `ArticlePriceImpact.zero()`. These represent missing data ‚Äî do NOT migrate them.
- **`article_impact_windows` idempotency key needs `UNIQUE INDEX`**: `ON CONFLICT (article_id, entity_id, window_type) DO NOTHING` requires a UNIQUE index, not a plain index. Use `CREATE UNIQUE INDEX`.
- **Ollama JSON mode**: Pass `"format": "json"` in the Ollama API request body for constrained JSON output. Prompt-only instruction is insufficient.
- **Score discontinuity is intentional**: Article may rank lower after day_t0 market data confirms no price movement vs. its pre-market LLM-predicted score. See PRD-0026 ¬ß7 AD-5.

## Chunk Text Retrieval (Option B ‚Äî MinIO, Wave A-1 extension)

`EnhancedChunkSearchUseCase._fetch_chunk_texts()` enriches search results with full chunk text:
1. Builds `key_map: dict[UUID, str]` ‚Äî only for results with `chunk_text_key` AND `granularity == "chunk"` (sections always fall back to `heading_path`).
2. Early-returns `{}` if `key_map` is empty (no I/O).
3. Checks Valkey cache at `nlp:v1:chunk_text:{chunk_id}` (1h TTL).
4. Calls `chunk_text_store.get_batch(uncached)` for cache misses only (skips call if `uncached` is empty).
5. Writes fetched texts to Valkey cache.
6. On `get_batch` failure: logs warning, falls back to `heading_path or ""` (never raises).
7. When `chunk_text_store is None`: text stays as `heading_path` ‚Äî backwards compatible.

Config: `NLP_PIPELINE_CHUNK_BUCKET` (default `"worldview"`), `NLP_PIPELINE_STORAGE_ACCESS_KEY` (empty = MinIO disabled, store stays None).

## Pitfalls (Wave C-4 additions)
- `FailureInfo[None]` has NO `value`/`key`/`headers` fields ‚Äî only `event_id`, `topic`, `partition`, `offset`, `attempt`, `last_error`, `record`
- `WatchlistCache(client=...)` expects raw `redis.asyncio.Redis`, NOT `ValkeyClient` ‚Äî pass `valkey._redis`
- `GLiNERLocalAdapter(model_path=..., semaphore=...)` ‚Äî no `model_id` kwarg
- `GLiNERHTTPAdapter(base_url=..., semaphore=...)` ‚Äî HTTP adapter for containerised GLiNER server
- `OllamaEmbeddingAdapter/ExtractionAdapter(base_url, model_id, semaphore)` ‚Äî all 3 required
- `_write_section_embeddings` / `_write_chunk_embeddings` now require `model_id` parameter
- Dispatcher uses custom `claim_batch` (not lease-based `BaseOutboxDispatcher`) ‚Äî incompatible schemas
- `process_message_from_failure` is a no-op for `FailureInfo[None]` ‚Äî original payload not recoverable
- **D-004 Eventual consistency gap**: Block 9 opens an `intel_sf()` session inside the outer `nlp_sf()` session. The intel_session commits before the nlp_session commit (~line 279 vs ~line 341). If the nlp commit fails, intel_db writes from entity resolution (Block 9) are already persisted and cannot be rolled back. Re-delivery will retry, potentially creating duplicate audit records. This is accepted eventual consistency. The `nlp_commit_failed_intel_writes_may_be_orphaned` warning log is emitted when the nlp commit fails.
- **GLiNER batching**: Block 4 (`run_ner_block`) calls `ner_client.batch_extract_entities(batch_inputs)` once per processing chunk ‚Äî all sections in a batch share a single model forward pass. NMS applied per-section after receiving the batch result.
- **UNNEST pg_trgm batch**: `batch_fuzzy_trigram` uses `UNNEST(:terms::text[]) AS q(search_term)` lateral join ‚Äî PostgreSQL casts the Python list to `text[]` automatically. Asyncpg rejects `::type` cast syntax in `text()` params ‚Äî use `bindparam('terms', type_=sa.ARRAY(sa.Text()))` if `text()` casts fail (BP-076).
