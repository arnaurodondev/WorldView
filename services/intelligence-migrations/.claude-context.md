# Intelligence Migrations (Init Container) — Agent Context

**Port**: None | **DB**: `intelligence_db` (DDL owner) | **Status**: Production-ready

## Mission
One-shot DDL owner for `intelligence_db`. Runs Alembic migrations, seeds reference data, and populates relation type embeddings. Exits after completion. This is an init container — NOT a long-running service.

## Key Operations (entrypoint.sh)
1. `alembic upgrade head` — creates all tables, indexes, partitions, seed data
2. Asserts `current revision == head` (exits non-zero on partial apply)
3. Runs all `seeds/*.sql` files (idempotent via `ON CONFLICT DO NOTHING`)
4. Runs `scripts/populate_embeddings.py` via Ollama — **non-blocking** (`|| true`)

## Critical Rule
This container is the ONLY source of DDL for `intelligence_db`. S6 (nlp-pipeline) and S7 (knowledge-graph) MUST set `ALEMBIC_ENABLED=false`.

## Pitfalls
- **Never add `intelligence_db` Alembic to S6 or S7** — migration chain conflict on next boot
- **`partition_key` columns are GENERATED ALWAYS AS STORED** — never include in INSERT statements (this is a DB invariant, not just a migration concern)
- **CONCURRENTLY is forbidden on partitioned tables** (PostgreSQL 16, BP-393) — use plain `CREATE INDEX` inside migrations
- **Embedding population is non-blocking** — if Ollama is unavailable, `populate_embeddings.py` exits 0; S7's `EmbeddingRefreshWorker` backfills async; Block 11 ANN soft-map returns no matches until backfill completes
- **Monthly partitions for `relation_evidence`, `claims`, `events`** — pre-seeded 2024-01 through 2026-12; S7's `MonthlyPartitionWorker` creates new months; add a migration only if S7 hasn't run yet and a new month's data arrives
- **Demo entity UUIDs are stable** — hardcoded in migration 0038; idempotent via `ON CONFLICT DO NOTHING`

## How to Add a Migration
```bash
alembic revision -m "describe_your_change"
```
Rules:
1. No `CONCURRENTLY` on partitioned tables (BP-393)
2. Add columns with defaults, never remove/rename (R12)
3. Never include `partition_key` in INSERT examples
4. Every `upgrade()` must have a working `downgrade()`
5. Document the PRD/DEF reference in the module docstring
