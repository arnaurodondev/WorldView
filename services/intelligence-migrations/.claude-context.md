# Intelligence Migrations (Init Container) — Agent Context

**Port**: None | **DB**: `intelligence_db` (DDL owner) | **Status**: Compliant

## Mission
One-shot DDL owner for `intelligence_db`. Runs Alembic migrations, seeds reference data, and populates relation type embeddings. Exits after completion. This is an init container, not a running service.

## Key Operations
- Applies DDL via `alembic upgrade head` (21 tables, 108+ partitions, 100+ indexes)
- Seeds `decay_class_config` (6 rows), `source_trust_weights` (11 rows), `relation_type_registry` (20 rows) inline in migration
- Seeds `model_registry` and `prompt_templates` via SQL scripts in `seeds/`
- Pre-seeds RANGE partitions for 2024-01 through 2026-12
- Populates `relation_type_registry.embedding` via Ollama (non-blocking on failure)

## Tables Owned
decay_class_config, source_trust_weights, model_registry, prompt_templates,
canonical_entities, entity_aliases, entity_embedding_state, llm_usage_log,
relation_type_registry, relations (HASH x8), relation_evidence_raw,
relation_evidence (RANGE monthly), relation_contradiction_links,
relation_summaries, claims (RANGE monthly), events (RANGE monthly),
event_entities, provisional_entity_queue, embedding_migration_state,
outbox_events, dead_letter_queue

## Critical Rule
This container is the ONLY source of DDL for `intelligence_db`. S6 (NLP Pipeline) and S7 (Knowledge Graph) must set `ALEMBIC_ENABLED=false` to prevent accidental migrations.

## Boot Order
Must complete BEFORE S6 and S7 start. Docker Compose uses `depends_on: condition: service_completed_successfully`.

## Env Vars
- `INTELLIGENCE_DB_URL` — required, Postgres connection string
- `EMBEDDING_BASE_URL` — Ollama API (default `http://ollama:11434`)
- `EMBEDDING_MODEL` — embedding model name (default `bge-large-en-v1.5`)

## Pitfalls
- Never add `intelligence_db` Alembic config to S6 or S7
- Monthly partitions for relation_evidence, claims, events must be created manually or via S7's `monthly_partition_job`
- `partition_key` columns are GENERATED ALWAYS AS STORED — never include in INSERT
- Embedding population is non-blocking — if Ollama is unavailable, embeddings are NULL until next run
