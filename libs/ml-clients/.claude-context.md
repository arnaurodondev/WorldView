# ml-clients — Agent Context

Shared ML-model client library: embedding, NER, extraction, description adapters plus
the **LLM cost/pricing** primitives every service uses to populate `llm_usage_log`.

## Layout
- `src/ml_clients/pricing.py` — **canonical single source of truth** for USD cost
  (`MODEL_PRICING`, `compute_cost` — Decimal). Also `is_priceable`,
  `LOCAL_FREE_MODELS`, `provider_cost_to_decimal` (PLAN-0117 W1).
- `src/ml_clients/cost.py` — legacy float estimator. `estimate_cost(provider, model_id,
  tin, tout)` now **delegates** to `pricing.compute_cost` (the `provider` arg is IGNORED,
  kept only for call-site compatibility). No independent price map lives here anymore.
- `src/ml_clients/usage_log.py` — `LlmUsageLogProtocol.log(...)` + `LlmCallUsage` value object.
- `src/ml_clients/adapters/` — DeepInfra (chat/description/embedding/extraction),
  OpenRouter, Gemini, Ollama, GLiNER adapters.

## Cost metering rules (PLAN-0117 — READ before touching any usage-log write)
- **DeepInfra returns `usage.estimated_cost`.** Capture it verbatim
  (`provider_cost_to_decimal(getattr(resp.usage, "estimated_cost", None))`) and PREFER it
  over the price matrix. `cost_source="provider"`. Adapters already do this for
  description / embedding / extraction.
- **Never log `$0` for a paid model.** If the provider omits a cost, fall back to
  `compute_cost(model_id, tin, tout)` → `cost_source="pricematrix"`. Zero is only correct
  for genuinely-local models (`LOCAL_FREE_MODELS`) with `cost_source="local"`.
- **`pricing.py` is the ONLY price map.** Do not reintroduce a second table. Add new
  models to `MODEL_PRICING` (dated `notes="as of YYYY-MM"`); `is_priceable` is the FR-7
  guardrail — a configured model that returns False would be a silent-zero.
- `model_id` casing/whitespace MUST match the provider's `model` string exactly or the
  matrix lookup silently returns `$0` (see the naming-convention note in `pricing.py`).
- Cost logging is **best-effort (NFR-1)** — parsing/logging must never raise into the
  request path; swallow + structlog-warn.

## Carrier / protocol fields (for S6/S7/S8/S9 wiring in W3/W4)
- `LlmCallUsage`: appended `provider_cost_usd: Decimal | None = None`,
  `cost_source: str = "pricematrix"` (frozen VO; positional constructions ending at
  `error_code` are unaffected — nothing in production constructs it today).
- `LlmUsageLogProtocol.log(...)`: appended kw-only `cost_source: str | None = None`,
  `user_id: UUID | None = None` (both default None → every existing call site compiles).

## Tests / validation
- Root venv: `source .venv/bin/activate` (needs `openai` — `[dev]` extra; install with
  `uv pip install "openai>=1.40,<2"` if missing).
- `python -m pytest tests/ -m "unit or not integration"` — 217 unit tests.
- `tests/integration/test_ollama_integration.py` needs a live Ollama at `:11434` (skip locally).
- Lint: `uvx ruff@0.4.0 check`/`format`; `mypy src --config-file ../../mypy.ini`.

## Pitfalls
- `openai` is lazily imported inside `DeepSeekExtractionAdapter.__init__`; without it,
  extraction tests ERROR at collection (module-level `import openai`).
- The embedding adapter parses raw httpx JSON (`data["usage"]["estimated_cost"]`) — the
  cost var is function-scoped + guarded so the error path (where `data` is unassigned)
  cannot NameError.
