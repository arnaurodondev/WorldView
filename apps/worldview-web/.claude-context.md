## Pitfalls
- `next.config.ts` rewrites strip `/api` prefix: `/api/v1/X` â†’ `API_GATEWAY_URL/v1/X`
- WebSocket needs `?token=` query param (browser WS API has no headers)
- `"use client"` directive required for any component using hooks, events, or browser APIs
- TanStack Query staleTime should be set per-query (not globally)
- `react-grid-layout` layout stored in localStorage for persistence
- **Portfolio localStorage keys (PLAN-0122)**: `worldview:portfolioMode:v1` (Simple\|Advanced detail level â€” also mirrored to the `?mode=` URL param via `hooks/usePortfolioMode.ts`), `worldview-holdings-cols` (AG-Grid column widths/order), `worldview:holdingsColGroups:v1` (Core/Portfolio/Advanced column-group visibility â€” orthogonal to widths/order), `worldview:portfolioTourSeen:v1` (onboarding-tour flag `pending`â†’`done`). The dual-mode `/portfolio` is a **render gate, never a fork** â€” Advanced output is byte-identical to the pre-0122 layout and guarded by `test_advanced_mode_is_todays_layout`. The Simple public default lives behind the `PORTFOLIO_SIMPLE_DEFAULT` constant in `lib/portfolio/mode-flag.ts` (rollback = flip to `false`).
- **AG Grid `rowHeight`**: portfolio components must use `rowHeight={20}` â€” `rowHeight=22` is now a forbidden pattern in the architecture test (PRD-0089 W2 lock). Screener components (`components/screener/`, `app/(app)/screener/`) likewise lock to 20 â€” see `__tests__/architecture/screener-row-height.test.ts` (PRD-0089 Wave I-A).
- **Screener categorical multi-select truncation (PRD-0089 IB-L1)**: L-1 backend takes first value of multi-select arrays only â€” see CountryFilterRow disclosure badge.
- **Screener fundamentals snapshot + credit-rating tone (PRD-0089 IB-L2)**: 6 opt-in numeric columns (AVG VOL / EPS / FCF / FCF MGN% / INT COV / ND/EBITDA) + "CREDIT RATING" badge shipped 2026-05-27 from Wave L-2 backend. The `creditRatingTone(rating)` helper in `lib/screener/credit-rating.ts` is the single source of truth for tierâ†’Terminal-Dark tone mapping: BBB-â†‘ positive, BB+..BB- warning, B+ and below + C/D negative; null/empty/undefined â†’ "muted" (renders as `text-muted-foreground` â€” QA #3 polish 2026-05-26: previously returned "negative" which painted unrated bonds red and misled compliance triage); unknown non-empty string still â†’ negative (parse failure). Callers MUST enumerate `text-positive/warning/negative/muted` explicitly â€” Tailwind JIT can't resolve `text-${tone}` dynamically. Multi-select `credit_ratings: string[]` round-trips intact via Wave L-2 IN-list (no truncation badge unlike IB-L1). FcfMarginCellRenderer heuristic |v|>1.5 falls back to assuming backend sent percent (not fraction) â€” defensive guard against silent contract drift.
- **Screener default-visible columns capped at 14** by PRD-0089 Â§6.3 density target; enforced by `lib/__tests__/screener-columns.test.ts` â€” to add a new default column, demote an existing one to opt-in in the same change.
- **Page-scoped hotkey pattern (no `useScopedHotkeys`)**: the codebase has `useChordHotkeys` and `useKeyboardShortcuts` but does NOT ship a `useScopedHotkeys` hook. Pages that need to override a global chord (e.g. screener `/` overriding the command palette) register a single `window.addEventListener("keydown", handler, { capture: true })` inside a `useEffect`. Capture phase fires before the global listener bubbling up, giving precise override semantics. Pattern documented in `app/(app)/screener/page.tsx` T-IA-09. Always guard letter chords against `INPUT`/`TEXTAREA`/`SELECT`/contentEditable targets so they don't fire while the user is typing.
