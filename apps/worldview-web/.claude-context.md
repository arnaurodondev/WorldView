## Critical Rules
- **Frontend → S9 only** — never call backend services directly
- **shadcn/ui only** — no other component libraries
- **Dark mode permanent** — `class="dark"` on `<html>`, never light mode
- **IBM Plex Mono for ALL numbers** — highest-impact visual rule
- **pnpm exact versions** — no `^` or `~` prefixes
- **Heavy inline comments** — user is new to Next.js; explain WHY not just WHAT

## Pitfalls
- `next.config.ts` rewrites strip `/api` prefix: `/api/v1/X` → `API_GATEWAY_URL/v1/X`
- WebSocket needs `?token=` query param (browser WS API has no headers)
- `"use client"` directive required for any component using hooks, events, or browser APIs
- TanStack Query staleTime should be set per-query (not globally)
- `react-grid-layout` layout stored in localStorage for persistence
- **AG Grid `rowHeight`**: portfolio components must use `rowHeight={20}` — `rowHeight=22` is now a forbidden pattern in the architecture test (PRD-0089 W2 lock). Screener components (`components/screener/`, `app/(app)/screener/`) likewise lock to 20 — see `__tests__/architecture/screener-row-height.test.ts` (PRD-0089 Wave I-A).
- **Screener categorical multi-select truncation (PRD-0089 IB-L1)**: L-1 backend takes first value of multi-select arrays only — see CountryFilterRow disclosure badge.
- **Screener fundamentals snapshot + credit-rating tone (PRD-0089 IB-L2)**: 6 opt-in numeric columns (AVG VOL / EPS / FCF / FCF MGN% / INT COV / ND/EBITDA) + CREDIT badge shipped 2026-05-27 from Wave L-2 backend. The `creditRatingTone(rating)` helper in `lib/screener/credit-rating.ts` is the single source of truth for tier→Terminal-Dark tone mapping: BBB-↑ positive, BB+..BB- warning, B+ and below + C/D negative; null/empty/unknown defaults to negative ("risk we can't classify"). Callers MUST enumerate `text-positive/warning/negative` explicitly — Tailwind JIT can't resolve `text-${tone}` dynamically. Multi-select `credit_ratings: string[]` round-trips intact via Wave L-2 IN-list (no truncation badge unlike IB-L1). FcfMarginCellRenderer heuristic |v|>1.5 falls back to assuming backend sent percent (not fraction) — defensive guard against silent contract drift.
- **Page-scoped hotkey pattern (no `useScopedHotkeys`)**: the codebase has `useChordHotkeys` and `useKeyboardShortcuts` but does NOT ship a `useScopedHotkeys` hook. Pages that need to override a global chord (e.g. screener `/` overriding the command palette) register a single `window.addEventListener("keydown", handler, { capture: true })` inside a `useEffect`. Capture phase fires before the global listener bubbling up, giving precise override semantics. Pattern documented in `app/(app)/screener/page.tsx` T-IA-09. Always guard letter chords against `INPUT`/`TEXTAREA`/`SELECT`/contentEditable targets so they don't fire while the user is typing.
